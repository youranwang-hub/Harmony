# zhangjiayuan STM32F407 小学期实验项目集合

本项目是基于 **STM32F407VE** 与 **QST-AU100 综合实验开发板** 整理的 STM32 小学期实验代码集合，包含基础实验、外围模块实验、拓展实验和最终综合实验。根目录 README 的作用是作为整个项目的总说明：说明有哪些实验、每个实验完成了什么、主要代码与外设在哪里、运行时应观察什么现象，以及这些实验如何逐步支撑最终综合项目。

最终综合实验为：

```text
综合实验：基于STM32F407的危险品运输与仓储安全监控系统
```

综合实验的具体工程架构、页面操作、串口命令、关键代码实现和完整验证流程已经在综合实验文件夹内单独整理。本根 README 只保留综合实验的项目级概览和关联说明，避免重复堆叠。

## 1. 项目总览

项目从 STM32F407 标准库空工程出发，逐步完成 GPIO、按键、中断、串口、定时器、PWM、ADC、DAC、DMA、RTC、I2C、SPI、CAN、LCD、触摸屏、GPS、指纹识别、NFC、红外等实验。实验之间不是孤立关系，而是从基础外设控制逐渐过渡到系统级综合应用。

统一环境如下：

| 项目 | 内容 |
|---|---|
| 开发板 | 青软 QST-AU100 综合实验开发板 |
| 主控芯片 | STM32F407VE |
| 开发工具 | MDK 5.25 / Keil uVision5 |
| 固件库 | STM32F4xx_DSP_StdPeriph_Lib_V1.4.0 |
| 工程类型 | STM32F4 标准外设库工程 |
| 下载调试器 | CMSIS-DAP |
| 下载接口 | SWD |
| 常用调试 | USART 串口、LCD 显示、LED、蜂鸣器、按键 |
| 代码要求 | 保持标准外设库，不引入 HAL |

学习路线可以理解为：

```text
工程模板与开发环境
  -> GPIO 输出和输入
  -> 中断、串口、定时器
  -> ADC、DAC、PWM 与模拟量处理
  -> DMA、RTC、I2C、SPI、CAN 等工程能力
  -> DHT11、MQ-2、GPS 等传感器
  -> LCD、XPT 触摸屏、NFC、FPR 等交互与认证
  -> 危险品运输与仓储安全监控综合实验
```

## 4. 基础控制实验

基础控制实验解决 STM32 程序对板载硬件的基本控制问题，是后续所有实验的底层基础。

### 4.1 LED 与蜂鸣器

- 目录：`【实验2-1】LED BEEP`
- 主要能力：GPIO 输出、LED 亮灭、蜂鸣器鸣叫。
- 关键代码：GPIO 时钟使能、输出模式配置、`GPIO_SetBits()` 和 `GPIO_ResetBits()` 控制电平。
- 运行现象：下载程序后 LED 按设定亮灭，蜂鸣器根据输出电平鸣叫或关闭。
- 注意事项：蜂鸣器可能是低电平有效，初始化时应先设置为关闭状态，避免复位后一直响。
- 与综合实验关系：用于系统状态灯、报警灯、操作成功短鸣和报警蜂鸣。

### 4.2 LED 流水灯

- 目录：`【实验2-2】LED流水灯`
- 主要能力：多路 GPIO 输出、延时控制、状态顺序切换。
- 关键代码：主循环中依次切换 LED 状态，并通过毫秒级延时形成流水效果。
- 运行现象：多个 LED 依次亮灭。
- 调试重点：延时时间过短会看不清变化，过长会影响响应速度。
- 与综合实验关系：开机自检、状态提示和报警闪烁都复用了“状态按周期切换”的思想。

### 4.3 按键驱动

- 目录：`【实验2-3】按键驱动`
- 主要能力：GPIO 输入读取、按键按下/释放判断、消抖。
- 关键代码：配置输入模式和上下拉状态，读取按键电平后延时确认。
- 运行现象：按下按键后 LED 或串口输出发生变化，松开后状态恢复或等待下一次触发。
- 调试重点：要区分按键有效电平，避免把未按下判断为按下。
- 与综合实验关系：用于报警状态下的实体按键消音和辅助操作。

### 4.4 外部中断

- 目录：`【实验3-1】外部中断`
- 主要能力：EXTI 线映射、NVIC 配置、中断服务函数。
- 关键代码：`SYSCFG_EXTILineConfig()`、`EXTI_Init()`、`NVIC_Init()` 和 `EXTIxx_IRQHandler()`。
- 运行现象：按键触发后即使主循环正在执行其他任务，也能进入中断服务函数。
- 调试重点：中断服务函数中必须清除中断挂起标志。
- 与综合实验关系：综合系统中按“报警消音/紧急响应”思想复用，不用于普通页面跳转。

## 5. 串口、定时与模拟量实验

这一类实验让系统具备可调试、可周期运行、可处理模拟量的能力。

### 5.1 USART 串口通信

- 目录：`【实验4-1】通信串行`
- 主要能力：USART 初始化、发送、接收、串口命令控制。
- 关键代码：配置波特率、TX/RX 复用引脚、接收缓冲区和命令解析。
- 运行现象：串口助手能接收到板端输出，发送命令后板端执行对应操作。
- 调试重点：波特率、数据位、停止位必须与串口助手一致。
- 与综合实验关系：综合系统保留串口作为调试和备用控制路径，用于页面切换、气体设定、场景触发、物资命令、日志导出和备用密码。

### 5.2 定时器

- 目录：`【实验5-1】时钟定时`
- 主要能力：TIM 定时中断、周期任务。
- 关键代码：预分频、自动重装值、中断使能和定时中断服务函数。
- 运行现象：LED 或串口信息按照固定周期变化。
- 调试重点：定时周期由定时器时钟、预分频和重装值共同决定。
- 与综合实验关系：用于触摸扫描、UI 刷新、数据采样、场景更新和日志周期任务。

### 5.3 PWM 呼吸灯

- 目录：`【实验5-2】呼吸灯`
- 主要能力：PWM 输出、占空比变化。
- 关键代码：定时器 PWM 模式配置，循环调整比较值。
- 运行现象：LED 亮度逐渐增强再减弱。
- 调试重点：PWM 频率和占空比变化速度需要匹配人眼观察。
- 与综合实验关系：用于风扇 25%、75%、100% 等占空比控制思路。

### 5.4 ADC

- 目录：`【实验6-1】ADC`
- 主要能力：模拟量采集、原始 ADC 值转换。
- 关键代码：ADC 通道配置、采样时间、转换启动和结果读取。
- 运行现象：旋转电位器后 ADC 值变化。
- 调试重点：通道必须与开发板实际连接一致。综合实验中电位器使用 `PC1 / ADC_Channel_11`。
- 与综合实验关系：用于 MQ-2、真实电位器、DAC 回读等气体数据源。

### 5.5 DAC

- 目录：`【实验6-2】DAC`
- 主要能力：模拟电压输出。
- 关键代码：DAC 通道初始化、数据对齐、输出值设置。
- 运行现象：DAC 引脚输出不同电压。
- 调试重点：DAC 输出不得超过 ADC 或外部输入允许范围。
- 与综合实验关系：用于生成气体模拟值，配合 ADC 形成 DAC-ADC 闭环演示。

## 6. 通信、时间与存储实验

这部分实验让工程具备批量数据传输、时间记录、参数保存和通信验证能力。

### 6.1 DMA 实验

- 目录：`（ARM基础实验）实验7_ STM32F407 DMA实验`
- 主要能力：DMA 自动搬运数据到 USART。
- 关键代码：DMA 数据流、通道、传输方向、内存地址、外设地址和传输长度配置。
- 运行现象：串口能接收到由 DMA 发送的数据。
- 调试重点：DMA 通道、USART 外设地址和完成标志必须匹配。
- 与综合实验关系：用于日志批量导出的设计依据。

### 6.2 RTC 实验

- 目录：`（ARM基础实验）实验8_ STM32F407 RTC实验`
- 主要能力：RTC 初始化、时间读取、时间显示。
- 关键代码：RTC 时钟源配置、时间日期结构体读写。
- 运行现象：串口或显示屏输出时间信息。
- 调试重点：RTC 时钟源和初始化流程必须正确。
- 与综合实验关系：用于运行时间、日志时间戳和事件顺序记录。

### 6.3 I2C 与 EEPROM

- 目录：`（ARM外围传感器实验）实验5_I2C与E2PROM实验`
- 主要能力：AT24C02 数据写入、读取和校验。
- 关键代码：I2C 起始、停止、应答、器件地址、存储地址、写周期等待。
- 运行现象：写入数据后再次读取能够得到相同内容。
- 调试重点：写入后要等待 EEPROM 内部写周期完成。
- 与综合实验关系：用于报警阈值和系统参数保存思路。

### 6.4 SPI 与 Flash

- 目录：`（ARM外围传感器实验）实验6_SPI与FLASH实验`
- 主要能力：W25Qxx ID 读取、页写、读回校验。
- 关键代码：SPI 初始化、片选控制、擦除、页写入、连续读取。
- 运行现象：读取到 Flash ID，写入数据后读回一致。
- 调试重点：写入前通常需要擦除，页写不能跨页随意写入。
- 与综合实验关系：用于日志记录和历史数据保存设计。

### 6.5 CAN 总线自发自收

- 目录：`（ARM外围传感器实验）实验7_CAN总线自发自收实验`
- 主要能力：CAN1 LoopBack 报文发送与接收。
- 关键代码：CAN 波特率、过滤器、发送邮箱、接收 FIFO。
- 运行现象：发送报文后能够在回环模式下接收到相同报文。
- 调试重点：过滤器配置不正确会导致接收不到报文。
- 与综合实验关系：综合系统使用 CAN LoopBack/单机演示模式表示通信状态，不把没有外部节点误判为真实在线。

## 7. 传感器实验

传感器实验让系统能够感知环境状态，并为综合实验中的监测和报警提供数据来源。

### 7.1 DHT11 温湿度

- 目录：`（ARM外围传感器实验）实验1_DHT11传感器实验`
- 主要能力：读取温度和湿度。
- 关键代码：单总线起始信号、响应检测、40 位数据读取、校验和判断。
- 运行现象：串口或显示屏显示温度、湿度。
- 调试重点：DHT11 对微秒级时序较敏感，延时函数必须可靠。
- 与综合实验关系：用于 MONITOR 页面环境信息和日志快照。

### 7.2 MQ-2 燃气传感器

- 目录：`（ARM外围传感器实验）实验2_MQ2燃气传感器实验`
- 主要能力：通过 ADC 读取气体传感器模拟输出。
- 关键代码：ADC 采集 MQ-2 AO 输出，按阈值判断正常或报警。
- 运行现象：气体模拟值变化时报警状态随之变化。
- 调试重点：MQ-2 需要预热，普通环境中数值可能不明显变化。
- 与综合实验关系：综合实验保留真实 MQ-2 路径，同时用电位器、DAC、场景和串口弥补实验条件限制。

### 7.3 GPS 定位

- 目录：`（ARM外围传感器实验）实验10_GPS定位实验`
- 主要能力：接收和解析 NMEA 报文。
- 关键代码：USART 接收 GPS 数据，解析 GGA/RMC 等字段中的状态、时间、经纬度。
- 运行现象：有定位时输出经纬度；无定位时显示无效状态。
- 调试重点：室内常常无法定位，应区分“串口通信正常”和“定位有效”。
- 与综合实验关系：综合系统采用 GPS REPLAY/LOST/ROUTE_DEV 方式演示，不因无定位触发危险报警。

## 8. 人机交互与身份识别实验

这部分实验让系统从串口调试扩展到 LCD 可视化界面、触摸操作和身份认证。

- 关键代码：FSMC/LCD 初始化、`LCD_Clear()`、字符输出和区域填充。

- 调试重点：LCD 初始化顺序、背光、坐标范围和颜色前景/背景要正确。

### 8.2 XPT 触摸屏

- 目录：`（ARM外围传感器实验）实验9_XPT触摸屏实验`
- 主要能力：XPT2046 触摸采样、坐标映射和区域命中。
- 关键代码：触摸原始坐标采样、校准参数、LCD 坐标转换。
- 运行现象：触摸屏幕不同区域，串口或 LCD 显示对应坐标或功能。
- 调试重点：必须保证“显示按钮区域”和“触摸命中区域”一致。
- 与综合实验关系：HOME 八宫格、各功能页面和报警页面均通过触摸操作。

### 8.3 FPR 指纹识别

- 目录：`（ARM外围传感器实验）实验11_FPR指纹识别实验`
- 主要能力：指纹录入、模板保存、搜索验证。
- 关键代码：串口命令帧、应答帧解析、两次录入和模板比对。
- 运行现象：放置手指后串口显示图像采集成功、录入成功或验证结果。
- 调试重点：指纹模块跳线、串口连接和供电模式必须正确。
- 与综合实验关系：用于 AUTH 页面注册验证、双重认证和物资操作确认。

### 8.4 NFC 读写卡

- 目录：`（ARM外围传感器实验）实验12_NFC读写卡实验`
- 主要能力：PN532 唤醒、读卡、UID 获取和读写验证。
- 关键代码：PN532 初始化、寻卡、UID 读取、认证和数据块读写。
- 运行现象：卡片靠近后串口显示卡 UID 或读写结果。
- 调试重点：卡片距离、模块接口模式和供电会影响识别。
- 与综合实验关系：用于身份注册、身份验证和 CARGO 物资确认。

### 8.5 IR 红外发送与接收

- 目录：`（ARM外围传感器实验）实验3_IR红外发送实验`、`（ARM外围传感器实验）实验4_IR红外接收实验`
- 主要能力：红外编码发送、红外按键值接收解析。
- 关键代码：载波/脉冲时序、引导码、数据码、按键码解析。
- 运行现象：发送端发出红外码，接收端识别并输出对应按键值。
- 调试重点：红外时序和接收头方向会影响效果。
- 与综合实验关系：红外是独立实验，最终综合系统未实际使用红外功能，因此不列入综合复用表。

## 11. 全项目验证建议

建议按从基础到综合的顺序验证，避免一开始就运行综合工程导致问题来源难以判断。

### 11.1 基础硬件验证

1. 先下载 LED/蜂鸣器实验，确认 GPIO 输出正常；
2. 再下载流水灯实验，确认延时和多 LED 控制正常；
3. 下载按键实验，确认按键输入和消抖正常；
4. 下载外部中断实验，确认按键中断能触发响应。

预期结果是 LED、蜂鸣器和按键现象都能被稳定观察。如果这里异常，应先检查开发板供电、工程型号、GPIO 引脚和下载配置。

### 11.2 串口和定时验证

1. 下载 USART 实验，确认串口助手能看到输出；
2. 输入简单命令，检查接收和解析是否生效；
3. 下载定时器和 PWM 实验，观察周期输出和呼吸灯效果；
4. 下载 DMA 实验，确认串口数据可批量发送。

这一步确认系统具备调试输出、命令输入和周期任务能力。

### 11.3 模拟量和传感器验证

1. ADC 实验中旋转电位器，确认 ADC 值变化；
2. DAC 实验中设置不同输出值，确认模拟输出；
3. DHT11 实验读取温湿度；
4. MQ-2 实验读取气体模拟量；
5. GPS 实验确认串口报文或无定位状态判断。

如果 MQ-2 或 GPS 数据不明显，不应视为实验失败，应结合真实采集、模拟输入和状态判断说明实验条件。

### 11.4 显示、触摸和身份验证

1. LCD 实验确认屏幕显示正常；
2. XPT 触摸屏实验确认坐标采样和区域命中正常；
3. FPR 实验确认指纹录入和验证流程；
4. NFC 实验确认 PN532 唤醒和卡 UID 读取。

综合实验中页面切换、认证和物资确认都依赖这些能力，若触摸不准，应先回到 XPT 基础实验检查校准。

### 11.5 综合实验验证

综合实验建议按以下顺序演示：

1. AUTH 注册和验证；
2. HOME 八宫格页面进入各模块；
3. MONITOR 查看环境和气体状态；
4. SOURCE 切换气体数据源；
5. SCENE 触发 GAS FAST 或 GPS LOST；
6. ALARM 页面观察蜂鸣器、LED、风扇和消音；
7. CARGO 执行物资加入、领出和 LIST；
8. LOG 导出日志；
9. SETTING 修改阈值并保存；
10. DEVICE 验证蜂鸣器、LED、风扇和 DAC。

详细按钮坐标、串口命令和期望输出请看综合实验目录内 README。

## 12. 编译下载说明

打开每个实验目录中的 Keil 工程文件后，应确认以下配置：

| 配置项 | 建议 |
|---|---|
| Device | STM32F407VE |
| C/C++ Define | `STM32F40_41xxx,USE_STDPERIPH_DRIVER` |
| Include Path | `USER`、`CORE`、`FWLIB/inc`、`BSP` 及实验实际目录 |
| Debug Driver | CMSIS-DAP Debugger |
| Interface | SWD |
| Flash Download | 使用匹配 STM32F407VE 的 Flash 算法 |

下载流程：

1. 打开开发板电源；
2. CMSIS-DAP 连接 SWD；
3. Keil 编译，确认无 error；
4. 点击 Download；
5. 输出窗口出现 Erase、Programming、Verify OK；
6. 按复位键观察实验现象。

## 13. 常见问题

### 13.1 CMSIS-DAP 报错

若出现 RDDI-DAP Error 或 No Device Found，优先检查：

- Debug Driver 是否选 CMSIS-DAP；
- Settings 中是否识别 SW Device；
- SWDIO、SWCLK、GND 是否连接正常；
- 开发板是否供电；
- USB 线是否接触良好。

### 13.2 下载成功但没有现象

下载成功只表示 Flash 写入成功，不表示程序一定运行到目标功能。建议先做 LED 最小程序，再逐步加入 LCD、串口和传感器。

### 13.3 LCD 不亮

先运行 LCD 基础实验。如果基础实验能亮，综合工程不亮，多半是初始化顺序或其他模块阻塞；如果基础实验也不亮，检查 LCD 驱动、背光、FSMC 和屏幕连接。

### 13.4 蜂鸣器一直响

检查蜂鸣器有效电平。若低电平有效，初始化后必须先输出关闭电平。综合实验中报警响和操作短鸣应分开处理。

### 13.5 GPS 无定位

室内无定位很常见。实验中应区分“GPS 串口通信正常”和“定位有效”。综合实验使用 REPLAY/LOST/ROUTE_DEV 演示，不虚构真实定位。

### 13.6 MQ-2 数值不变化

MQ-2 需要预热且普通空气变化小。可以通过电位器、DAC 或串口模拟气体变化，综合实验中统一映射到 0~1000 后进入报警逻辑。

### 13.7 触摸点不到按钮

查看串口 `[TOUCH] x=... y=...` 输出。如果有坐标但 UI 不响应，说明坐标没落在按钮区域；如果没有坐标，检查 XPT 初始化和触摸连接。

## 14. 学习路线总结

整个项目体现了从单个外设到综合系统的递进过程：

| 阶段 | 重点能力 | 对应实验 |
|---|---|---|
| 基础控制 | GPIO 输出、输入、中断 | LED、蜂鸣器、流水灯、按键、外部中断 |
| 调试与调度 | 串口、定时器、PWM、DMA | USART、TIM、呼吸灯、DMA |
| 模拟与传感器 | ADC、DAC、温湿度、气体、GPS | ADC、DAC、DHT11、MQ-2、GPS |
| 通信与存储 | I2C、SPI、CAN、RTC | EEPROM、Flash、CAN、RTC |
| 人机交互 | LCD、触摸屏 | LCD、XPT |
| 身份识别 | 指纹、NFC | FPR、NFC |
| 系统集成 | 场景、报警、库存、日志 | 危险品运输与仓储综合实验 |

通过这些实验，可以逐步掌握 STM32F407 标准库工程搭建、外设初始化、模块驱动迁移、串口调试、LCD 触摸交互、身份认证、传感器数据处理和系统级状态管理。综合实验作为最终成果，将前面的实验能力组合成一个具有真实应用背景的嵌入式安全监控系统。
