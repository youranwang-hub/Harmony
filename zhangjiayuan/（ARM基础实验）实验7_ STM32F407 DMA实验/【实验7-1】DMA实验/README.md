# STM32F407 DMA 实验说明

本实验对应 `【实验7-1】DMA实验`，代码位于 `代码` 文件夹中，包含基础版 `main.c` 和拓展版 `7mainplus1.c`。实验平台为 STM32F407，使用 USART2 与 DMA 完成串口数据发送、接收和重复回显。

## 实验目的

- 理解 DMA 在内存与外设之间搬运数据的工作机制。
- 掌握 USART2 与 DMA1_Stream6、DMA1_Stream5、Channel 4 的对应关系。
- 完成 USART2 使用 DMA 发送固定字符串的基础任务。
- 在基础任务上增加 DMA 接收、USART2 空闲中断、命令解析和发送完成中断。

## 硬件连接

| 模块 | 引脚/资源 | 说明 |
| --- | --- | --- |
| USART2_TX | PA2 | 串口发送，连接上位机串口接收端 |
| USART2_RX | PA3 | 串口接收，连接上位机串口发送端 |
| DMA 发送 | DMA1_Stream6 / Channel 4 | USART2_TX，内存到外设 |
| DMA 接收 | DMA1_Stream5 / Channel 4 | USART2_RX，外设到内存 |

串口参数：115200 bit/s，8 位数据位，1 位停止位，无校验，无硬件流控。

## 代码结构

```text
代码
├─main.c          # 基础版：DMA 单次发送 Hello Everyone!
└─7mainplus1.c    # 拓展版：DMA 接收命令并按次数回显
```

## 核心流程

### 基础版

1. 初始化 GPIOA、USART2 和 DMA1。
2. 将 PA2、PA3 配置为 USART2 复用功能。
3. 配置 DMA1_Stream6，外设地址指向 `USART2->DR`。
4. 将 `SendBuf` 中的 `"Hello Everyone!\r\n"` 通过 DMA 发送到 USART2。
5. DMA 普通模式发送完成后停止，主循环空转。

### 拓展版

1. 初始化 USART2、DMA 接收流和 DMA 发送流。
2. DMA1_Stream5 接收串口数据到 `RxBuf`。
3. USART2 空闲中断判断一帧命令接收完成。
4. 解析 `次数:内容` 命令，例如 `8:ok`。
5. 将待发送内容复制到 `TxBuf`，启动 DMA1_Stream6 发送。
6. DMA 发送完成中断递减剩余次数，直到指定次数发送完成。

## 运行现象

- 基础版：先打开 QCOM 串口，再复位开发板，串口接收区出现一次 `Hello Everyone!`。
- 拓展版：串口发送 `8:ok` 后，接收区连续出现 8 行 `ok`。


## 关键参数

| 参数 | 值 |
| --- | --- |
| 接收缓冲区 | 128 字节 |
| 发送缓冲区 | 128 字节 |
| 最大重复次数 | 1000 |
| 命令格式 | `次数:内容`，冒号为英文半角 |

## 注意事项

- DMA 普通模式只发送一次，验证基础版时应先打开串口工具，再复位开发板。
- 拓展版命令必须使用英文半角冒号。
- 重复发送期间程序会保持发送忙状态，此时新命令不会覆盖当前发送缓冲区。
- 如果编译提示缺少头文件，应检查 Keil 工程包含路径和当前工程模板配置。
