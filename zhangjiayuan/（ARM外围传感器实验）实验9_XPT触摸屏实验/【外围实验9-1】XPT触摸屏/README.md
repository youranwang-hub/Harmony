# 实验9-1 XPT2046触摸屏实验

## 实验内容

本实验使用 STM32F407 通过 GPIO 软件 SPI 读取 XPT2046 电阻触摸屏控制器，实现触摸按下、坐标采样、坐标校准和松手检测。当前代码将触摸事件通过 USART2 打印出来，用于验证触摸屏基础输入功能。

## 代码结构

- `代码/USER/main.c`：实验主程序，初始化串口和 XPT2046，并循环检测触摸状态。
- `代码/BSP/xpt2046.c`、`代码/BSP/xpt2046.h`：XPT2046 GPIO 软件 SPI、坐标采样、校准和状态检测。
- `代码/BSP/usart.c`、`代码/BSP/usart.h`：USART2 调试串口。

## 硬件与接口

- 触摸控制器：XPT2046
- 通信方式：GPIO 模拟 SPI
- CS：PD13
- CLK：PE0
- MOSI：PE2
- MISO：PE3
- IRQ：PE4
- 调试串口：USART2，波特率 115200

## 程序流程

1. 初始化延时和 USART2。
2. 调用 `XPT2046_Init()` 初始化触摸屏控制引脚。
3. 串口打印 `XPT2046 TEST`。
4. 主循环中调用 `XPT2046_DetectTouch()` 检测触摸状态。
5. 每次循环延时 50 ms，降低抖动和串口输出频率。

## 触摸检测逻辑

驱动通过 XPT2046 的 X、Y 通道命令读取 12 位 ADC 原始值，再根据校准系数转换为屏幕坐标。`XPT2046_DetectTouch()` 内部维护按下和松开的状态变化，并通过弱函数回调输出事件：

- `XPT2046_TouchDown()`：检测到按下时打印坐标。
- `XPT2046_TouchUp()`：检测到松手时打印坐标。

## 运行现象

下载程序后，在串口终端中可以看到实验启动信息。手指或触控笔按压屏幕时，串口打印按下坐标；松开时打印松手坐标。若坐标随触摸位置变化，说明触摸屏采样和校准流程工作正常。

## 注意事项

- 若一直无触摸输出，优先检查 IRQ 引脚 PE4 和 CS 引脚 PD13。
- 电阻屏采样容易受抖动影响，驱动中已做基础状态判断；精度要求更高时可增加多次采样滤波。
