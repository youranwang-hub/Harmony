# 实验4 IR 红外接收实验

本实验使用 STM32F407 的 TIM9 输入捕获功能接收红外遥控信号，并按照 NEC 协议解析地址码和键码，通过 USART2 输出解码结果。

## 代码结构

```text
代码/
├── BSP/
│   ├── ir_recv.c / ir_recv.h    # 红外接收、输入捕获、NEC 解码
│   └── usart.c / usart.h        # USART2 调试串口
├── USER/
│   └── main.c                   # 主程序
└── UTILS/
    ├── delay.c / delay.h
    └── utils.c / utils.h
```

## 硬件与外设

| 功能 | 外设/引脚 | 说明 |
| --- | --- | --- |
| 红外接收 | PE5 / TIM9_CH1 | 捕获红外接收头输出的下降沿 |
| 调试串口 | USART2，PA2/PA3 | 115200，8N1，用于 `printf` 输出 |

## 主要实现

`IR_Recv_Init()` 完成红外接收初始化：

- `IR_GPIO_Init()` 将 PE5 配置为 TIM9_CH1 复用输入，上拉。
- `IR_TIM_Init()` 将 TIM9 配置为输入捕获和更新溢出中断。
- TIM9 计数频率设置为 1 MHz，捕获值单位约为 us。
- TIM9 最大计时 20 ms，用于判断接收超时并清理残缺帧。

`TIM1_BRK_TIM9_IRQHandler()` 是核心中断函数：

- 捕获 TIM9_CH1 下降沿。
- 读取两个下降沿之间的时间间隔。
- 识别 NEC 完整引导码，范围约 12.5 ms 到 14.5 ms。
- 识别 NEC 重复码，范围约 10.5 ms 到 12 ms。
- 根据下降沿间隔判断数据 0 或数据 1。
- NEC 数据按低位先发，接收满 32 位后保存到 `IR_FrameData`。

`IR_Recv()` 在主循环中执行：

1. 判断是否已有完整 32 位数据。
2. 关闭中断并复制接收数据，避免中断与主循环同时访问。
3. 拆分 4 个字节：地址码、地址反码或扩展地址高字节、键码、键码反码。
4. 打印完整原始数据。
5. 校验键码和键码反码。
6. 输出地址码和键码。

## 主程序流程

`USER/main.c` 的流程：

1. 初始化 USART2，波特率 115200。
2. 初始化红外接收模块。
3. 打印 `IRRecv Init OK`。
4. 在 `while(1)` 中持续调用 `IR_Recv()` 解析已接收的完整红外数据。

## 运行现象

程序启动后串口输出：

```text
IRRecv Init OK
```

按下红外遥控器按键后，如果收到合法 NEC 帧，串口会输出类似：

```text
IRRecv:[XXXXXXXX]
IRRecv:addr:1, code:123
```

如果键码与键码反码校验失败，会输出：

```text
IRRecv:code check error
```

## 使用方法

1. 将 `代码/BSP`、`代码/USER`、`代码/UTILS` 加入 STM32F407 标准外设库工程。
2. 配置工程头文件路径。
3. 编译下载到开发板。
4. 打开串口调试助手，设置 115200、8N1。
5. 使用红外遥控器对准接收头发送信号，观察串口解码输出。

## 注意事项

- 本程序按 NEC 协议时序判断红外数据，非 NEC 协议遥控器可能出现校验失败或无法解析。
- 接收头空闲时通常为高电平，收到红外载波时输出低电平，因此程序捕获下降沿。
- 程序只在完整接收 32 位后才交给主循环解析，避免把残缺帧当成有效数据。
