# 实验7-1拓 CAN总线拓展实验

## 实验内容

本拓展实验在 CAN1 回环收发的基础上，增加标准帧和扩展帧交替发送、发送数据自动递增、串口命令修改发送间隔等功能。程序通过 USART2 输出当前发送帧类型、CAN ID 和数据内容，便于观察两种 CAN 帧格式的差异。

## 代码结构

- `代码/USER/main.c`：拓展主程序，组织标准帧和扩展帧交替发送。
- `代码/BSP/can.c`、`代码/BSP/can.h`：CAN1 初始化、扩展帧发送和接收中断。
- `代码/BSP/usart.c`、`代码/BSP/usart.h`：USART2 串口输出及命令输入相关支持。

## 与基础版的区别

- 基础版只发送扩展帧，拓展版设计为标准帧和扩展帧交替发送。
- 标准帧 ID 使用 `0x123`，扩展帧 ID 使用 `0x1234`。
- 标准帧数据从 `11 22 33 44 55 66 77 88` 开始，扩展帧数据从 `88 77 66 55 44 33 22 11` 开始。
- 每次发送后，数据数组中的字节会自动加 1，方便在串口中观察数据变化。
- 启动提示中给出串口命令：输入 `1`、`3`、`5` 可切换发送间隔为 1 秒、3 秒、5 秒。

## 程序流程

1. 初始化延时、USART2 和 CAN1。
2. 串口打印 `CAN LoopBack Test` 以及间隔设置说明。
3. 主循环先发送标准数据帧，再发送扩展数据帧。
4. 发送完成后分别递增标准帧数据和扩展帧数据。
5. 根据 `g_can_send_interval` 延时后进入下一轮发送。

## 运行现象

打开 115200 波特率串口终端后，可以看到标准帧和扩展帧交替输出；CAN1 处于回环模式时，发送帧也会被接收中断打印出来。输入 `1`、`3` 或 `5` 后，后续发送间隔应切换到对应秒数。
