# 实验7-1 CAN总线自发自收实验

## 实验内容

本实验基于 STM32F407 的 CAN1 控制器完成 CAN 总线自发自收测试。当前代码将 CAN1 配置为 LoopBack 回环模式，通过 USART2 打印发送和接收结果，用于验证 CAN 初始化、发送邮箱、接收 FIFO、中断回调和帧数据解析流程。

## 代码结构

- `代码/USER/main.c`：实验主程序，初始化延时、USART2 和 CAN1，并周期发送 CAN 数据帧。
- `代码/BSP/can.c`、`代码/BSP/can.h`：CAN1 GPIO、位时序、过滤器、发送函数和接收中断处理。
- `代码/BSP/usart.c`、`代码/BSP/usart.h`：USART2 调试串口，支持 `printf` 输出。

## 硬件与接口

- 主控：STM32F407
- CAN 外设：CAN1
- CAN 引脚：PA11、PA12，复用为 CAN1 功能
- 调试串口：USART2，波特率 115200
- CAN 模式：LoopBack 回环模式，便于不接外部 CAN 节点时进行自发自收测试

## 程序流程

1. 调用 `delay_init(168)` 初始化系统延时。
2. 调用 `UART2_Init(115200)` 初始化调试串口。
3. 调用 `CAN1_Init()` 初始化 CAN1、过滤器和 FIFO0 接收中断。
4. 主循环中每 5 秒调用 `CAN1_SendMsg(0x1234, pTxData, 8)` 发送 8 字节扩展帧。
5. CAN1 接收到回环数据后进入 `CAN1_RX0_IRQHandler()`，打印 CAN ID 和数据内容。

## 当前发送数据

主程序发送的 8 字节数据为：

```text
88 77 66 55 44 33 22 11
```

发送 ID 为 `0x1234`，代码中使用扩展帧格式发送。

## 运行现象

下载程序后，串口终端会周期性看到发送数据打印；由于 CAN1 工作在回环模式，发送的数据会被本机接收，并在接收中断中打印接收到的 CAN ID 和 8 字节数据。若发送和接收打印内容一致，说明 CAN 初始化、发送、接收和中断流程正常。

## 注意事项

- 当前实验重点是 CAN 控制器内部回环验证，不依赖另一块开发板。
- 若切换为 Normal 正常模式，需要连接 CAN 收发器并正确接入 CANH、CANL 总线。
- CAN 位时序在 `can.c` 中配置，当前 Prescaler 为 6，BS1 为 6tq，BS2 为 7tq。
