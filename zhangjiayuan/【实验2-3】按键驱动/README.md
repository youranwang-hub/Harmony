# 实验2-3 按键驱动

本实验使用 GPIO 输入读取四个按键状态，并通过按键控制三盏 LED。

## 文件说明

| 文件 | 功能 |
| --- | --- |
| `main.c` | 轮询读取 KEY1 至 KEY4，按下后立即执行对应 LED 控制 |
| `2mainplus3.c` | 拓展实验，确认按键按下后等待释放，释放后再执行对应 LED 控制 |

## 硬件连接

| 外设 | 引脚 | 说明 |
| --- | --- | --- |
| KEY1 | PA0 | 上拉输入，按下为低电平 |
| KEY2 | PC13 | 上拉输入，按下为低电平 |
| KEY3 | PC6 | 上拉输入，按下为低电平 |
| KEY4 | PC7 | 上拉输入，按下为低电平 |
| LED1 | PB5 | 低电平点亮 |
| LED2 | PB0 | 低电平点亮 |
| LED3 | PB1 | 低电平点亮 |

## 程序逻辑

`main.c` 的主要流程：

1. 使能 GPIOA 和 GPIOC 时钟。
2. 将 PA0、PC13、PC6、PC7 配置为上拉输入。
3. 使能 GPIOB 时钟。
4. 将 PB5、PB0、PB1 配置为推挽输出，并默认输出高电平熄灭 LED。
5. 在主循环中轮询四个按键输入电平。
6. 检测到低电平后延时 20 ms 进行按键消抖。
7. 消抖后再次读取按键，并执行对应动作。

按键功能：

| 按键 | 动作 |
| --- | --- |
| KEY1 | 点亮 LED1 |
| KEY2 | 点亮 LED2 |
| KEY3 | 点亮 LED3 |
| KEY4 | 熄灭 LED1、LED2、LED3 |

`2mainplus3.c` 增加了按键释放检测：确认某个按键按下后，程序会停留在 `while` 循环中等待该按键恢复高电平，再延时 20 ms 消除抬起抖动，最后执行 LED 控制。

## 运行现象

运行基础程序时，按 KEY1、KEY2、KEY3 可以分别点亮对应 LED，按 KEY4 可以关闭三盏 LED。拓展程序的效果相同，但动作发生在按键释放之后，更适合避免一次按压被重复触发。

## 使用方法

将 `main.c` 或 `2mainplus3.c` 作为工程入口文件编译下载，观察按键与 LED 状态变化。
