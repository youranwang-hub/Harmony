# 实验4-1 通信串行

本实验使用 USART2 与上位机串口调试助手通信，通过接收指令控制 LED 和蜂鸣器。

## 文件说明

| 文件 | 功能 |
| --- | --- |
| `main.c` | 基础串口指令控制，支持 `BEEP`、`LED1`、`LED2`、`LED3` |
| `4mainplus1.c` | 拓展实验，支持单个 LED 开关和流水灯启停 |

## 硬件连接与串口参数

| 项目 | 配置 |
| --- | --- |
| USART | USART2 |
| TX | PA2 |
| RX | PA3 |
| 波特率 | 115200 |
| 数据位 | 8 bit |
| 停止位 | 1 bit |
| 校验 | 无 |
| 流控 | 无 |
| 帧结束 | 回车换行 `\r\n` |

其他外设：

| 外设 | 引脚 | 有效电平 |
| --- | --- | --- |
| LED1 | PB5 | 低电平点亮 |
| LED2 | PB0 | 低电平点亮 |
| LED3 | PB1 | 低电平点亮 |
| 蜂鸣器 | PA8 | 低电平响，仅 `main.c` 使用 |

## 基础程序指令

`main.c` 初始化 USART2 后发送：

```text
Hello Everyone!
```

接收中断 `USART2_IRQHandler()` 逐字节接收数据，以 `\r\n` 判断一帧结束。主循环检测 `Rx_Frame_Flag` 后解析指令。

支持指令：

| 指令 | 功能 |
| --- | --- |
| `BEEP` | 蜂鸣器响约 50 ms 后关闭，并返回“蜂鸣器” |
| `LED1` | 点亮 LED1 约 200 ms 后熄灭，并返回 `LED1` |
| `LED2` | 点亮 LED2 约 200 ms 后熄灭，并返回 `LED2` |
| `LED3` | 点亮 LED3 约 200 ms 后熄灭，并返回 `LED3` |

收到不符合格式的指令时，程序返回“指令错误！”。

## 拓展程序指令

`4mainplus1.c` 支持更完整的 LED 控制：

| 指令 | 功能 |
| --- | --- |
| `LED1_ON` | 点亮 LED1 |
| `LED1_OFF` | 熄灭 LED1 |
| `LED2_ON` | 点亮 LED2 |
| `LED2_OFF` | 熄灭 LED2 |
| `LED3_ON` | 点亮 LED3 |
| `LED3_OFF` | 熄灭 LED3 |
| `FLOW_ON` | 开启 LED1 -> LED2 -> LED3 流水灯 |
| `FLOW_OFF` | 关闭流水灯并熄灭所有 LED |

拓展程序使用 `Command_Match()` 判断完整指令是否匹配，使用 `Flow_Flag` 控制是否持续运行流水灯。流水灯每 200 ms 前进一步。

## 使用方法

1. 将需要运行的源码作为工程入口 `main.c`。
2. 编译并下载到开发板。
3. 打开串口调试助手，选择 CH340 对应串口。
4. 设置 `115200, 8N1`。
5. 发送指令时开启回车换行，或手动发送 `\r\n`。
