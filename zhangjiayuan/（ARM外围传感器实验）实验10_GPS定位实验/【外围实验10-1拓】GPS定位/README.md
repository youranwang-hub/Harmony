# 实验10-1拓 GPS定位拓展实验

## 实验内容

本拓展实验在 GPS NMEA 基础解析上增加更适合直接阅读的定位信息处理，包括经纬度十进制度换算、UTC 时间转换为北京时间、节速转换为 km/h、运动状态判断和定位有效性提示。主程序入口与基础版基本一致，主要拓展集中在 GPS 解析模块中。

## 代码结构

- `代码/USER/main.c`：初始化 USART2 和 GPS，循环调用 `GPS_ReadAndParse()`。
- `代码/BSP/gps.c`、`代码/BSP/gps.h`：UART4 接收和 NMEA 数据帧处理。
- `代码/BSP/gps_parser.c`、`代码/BSP/gps_parser.h`：拓展后的 NMEA 字段解析和定位信息换算。
- `代码/BSP/usart.c`、`代码/BSP/usart.h`：USART2 调试输出。

## 与基础版的区别

- 基础版主要打印 NMEA 原始字段。
- 拓展版将纬度、经度从 NMEA 的度分格式转换为十进制度。
- 将 UTC 时间转换为北京时间。
- 将 RMC/VTG 中的节速换算为 km/h，换算系数为 `1 knot = 1.852 km/h`。
- 根据速度判断运动状态，例如静止、低速移动或移动中。
- 对定位状态进行有效或无效提示，方便快速判断当前 GPS 数据是否可用。

## 硬件与接口

- GPS 通信：UART4，波特率 9600
- 调试输出：USART2，波特率 115200
- UART4 引脚：PC10、PC11
- 模块跳线：需切换到 GPS 侧

## 程序流程

1. 初始化系统延时和 USART2。
2. 初始化 GPS UART4 接收。
3. 接收 GPS 模块输出的 NMEA 数据。
4. 按语句类型解析 GGA、RMC、GLL、GSA、GSV、VTG。
5. 对关键字段进行单位转换、时间转换和状态判断。
6. 通过 USART2 输出更直观的定位结果。

## 运行现象

串口终端除原始字段外，会显示换算后的十进制经纬度、北京时间、km/h 速度和运动状态。若 GPS 尚未定位，程序会提示定位无效或显示无效状态；成功定位后可看到稳定的经纬度和速度信息。

## 注意事项

- 拓展功能依赖 NMEA 语句中有效的状态字段和速度字段，定位无效时换算结果只作为参考。
- GPS 室内定位不稳定，建议在可见天空较多的位置测试。
