# 实验10-1 GPS定位实验

## 实验内容

本实验使用 STM32F407 通过 UART4 接收 GPS 模块输出的 NMEA 0183 数据，并在 USART2 串口中打印原始定位语句和解析结果。当前代码支持对 GGA、RMC、GLL、GSA、GSV、VTG 等常见 NMEA 语句进行字段解析。

## 代码结构

- `代码/USER/main.c`：实验主程序，初始化 USART2 和 GPS，并循环读取解析数据。
- `代码/BSP/gps.c`、`代码/BSP/gps.h`：UART4 接收、缓冲区管理、空闲中断帧结束判断和解析入口。
- `代码/BSP/gps_parser.c`、`代码/BSP/gps_parser.h`：NMEA 语句分类和字段解析。
- `代码/BSP/usart.c`、`代码/BSP/usart.h`：USART2 调试输出。

## 硬件与接口

- GPS 模块：LC86L 或兼容 NMEA 输出模块
- GPS 通信串口：UART4，波特率 9600
- UART4 引脚：PC10、PC11
- 调试串口：USART2，波特率 115200
- 注意：GPS 模块与 Zigbee 可能共用跳线，实验时需将跳线切换到 GPS 侧。

## 程序流程

1. 初始化延时和 USART2。
2. 调用 `GPS_Init(9600)` 初始化 UART4 接收。
3. 串口打印 `GPS Test`。
4. 主循环中调用 `GPS_ReadAndParse()`。
5. UART4 接收到一帧 NMEA 数据后，程序打印 `GPS DATA` 并逐条调用 `gps_parse()`。

## 解析内容

当前解析器按语句类型拆分字段，主要包括：

- `GGA`：UTC 时间、纬度、经度、定位质量、卫星数、高度等。
- `RMC`：时间、定位状态、纬度、经度、速度、日期等。
- `GLL`：经纬度、UTC 时间和状态。
- `GSA`：定位模式、定位类型和 DOP 信息。
- `GSV`：可见卫星信息。
- `VTG`：航向和地面速度信息。

## 运行现象

GPS 模块输出有效 NMEA 数据后，串口终端会显示原始 GPS 数据和解析后的字段。若在室内收不到有效定位，仍可能看到部分 NMEA 语句，但定位状态、卫星数或经纬度可能为空或无效。

## 注意事项

- GPS 首次定位需要一定时间，建议在靠近窗边或室外测试。
- 若串口没有输出，检查 GPS 供电、UART4 引脚、波特率 9600 和跳线方向。
- 当前基础版以字段拆解和打印为主，经纬度换算、北京时间转换等功能在拓展版中实现。
