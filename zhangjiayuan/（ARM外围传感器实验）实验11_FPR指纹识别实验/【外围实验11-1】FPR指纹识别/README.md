# 实验11-1 FPR指纹识别实验

## 实验内容

本实验使用 STM32F407 通过 USART1 控制 ZN632 指纹识别模块，实现指纹录入和指纹匹配。程序通过 USART2 提供串口菜单，用户可选择添加新指纹或进行指纹匹配，适合验证指纹模块通信协议和基础识别流程。

## 代码结构

- `代码/USER/main.c`：实验主程序，显示操作菜单并调用指纹录入或匹配功能。
- `代码/BSP/fpr.c`、`代码/BSP/fpr.h`：ZN632 指纹模块通信、命令封包、应答解析和业务流程。
- `代码/BSP/usart.c`、`代码/BSP/usart.h`：USART2 调试输出。

## 硬件与接口

- 指纹模块：ZN632
- 指纹模块串口：USART1
- 调试串口：USART2，波特率 115200
- 模块供电控制：PC9
- 指纹模块通信波特率：代码中调用 `FPR_Init(56700)`
- 注意：指纹模块和 NFC 模块可能共用 USART1 跳线，实验时需切换到指纹模块侧。

## 菜单功能

串口菜单包含两个选项：

```text
1 Add New Finger
2 Match Finger
```

- `1`：添加新指纹，按模块流程采集图像、生成特征、合成模板并存入指纹库。
- `2`：匹配指纹，采集当前手指图像并在指纹库中搜索匹配项。

## 程序流程

1. 初始化延时和 USART2。
2. 调用 `FPR_Init(56700)` 初始化指纹模块通信和供电。
3. 调用 `ZN632_VryPwd()` 校验模块通信密码。
4. 调用 `ZN632_ReadIndexTable()` 读取指纹库索引表。
5. 循环显示菜单，根据串口输入执行录入或匹配。

## 运行现象

串口终端显示 `FPR Init OK` 后进入菜单。选择录入时，根据提示按压手指完成两次采集并保存模板；选择匹配时，按压已录入手指后串口会输出匹配结果，未匹配时会提示失败信息。

## 注意事项

- 指纹采集时手指需要稳定覆盖传感器，过快移开会导致图像生成失败。
- 若初始化失败，优先检查 USART1 跳线、模块供电、波特率和 PC9 控制引脚。
- 当前基础版只提供录入和匹配两个核心功能，库容量统计和按 ID 删除在拓展版中实现。
