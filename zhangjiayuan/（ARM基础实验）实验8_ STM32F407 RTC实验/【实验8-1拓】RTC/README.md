# STM32F407 RTC 拓展实验说明

本实验对应 `【实验8-1拓】RTC`，代码位于 `代码` 文件夹中。拓展版在 RTC 时间日期显示基础上加入按键设置、闹钟保存、闹钟中断、LED 和蜂鸣器报警。

## 实验目的

- 在 RTC 基础显示功能上实现人机交互设置。
- 掌握按键扫描、模式切换和数值编辑流程。
- 学会配置 RTC Alarm，实现定时闹钟触发。
- 将 OLED、按键、LED、蜂鸣器与 RTC 综合使用。

## 硬件连接

| 模块 | 引脚/资源 | 说明 |
| --- | --- | --- |
| OLED_SCLK | PB6 | OLED 时钟控制线 |
| OLED_SDIN | PB7 | OLED 数据控制线 |
| KEY1 | PA0 | 进入设置/切换设置项 |
| KEY2 | PC13 | 当前设置项加 1 |
| KEY3 | PC6 | 当前设置项减 1 |
| KEY4 | PC7 | 保存设置 |
| LED1 | PB5 | 闹钟提示 |
| BEEP | PA8 | 蜂鸣器，低电平响 |
| RTC Alarm | RTC 内部闹钟 | 触发中断并置位报警标志 |

## 代码结构

```text
代码
├─BSP
│  ├─bsp_io.c / bsp_io.h     # 按键、LED、蜂鸣器
│  ├─delay.c / delay.h       # 延时
│  ├─dht11.c / dht11.h       # 保留的传感器驱动
│  ├─oled.c / oled.h         # OLED 显示
│  ├─oledfont.h              # OLED 字库
│  └─rtc.c / rtc.h           # RTC 与闹钟相关函数
└─USER
   └─main.c                  # 设置流程、闹钟处理、页面显示
```

## 核心流程

1. 初始化按键、LED、蜂鸣器、OLED 和 RTC。
2. 读取或设置 RTC 当前时间日期。
3. 读取已保存的闹钟配置并重新设置 RTC Alarm。
4. 正常模式下 OLED 显示时间和日期。
5. 按 KEY1 进入设置模式，依次编辑时、分、秒、年、月、日、闹钟小时、闹钟分钟和闹钟开关。
6. KEY2/KEY3 调整数值，KEY4 保存修改。
7. 闹钟触发后进入报警处理，LED 和蜂鸣器周期动作。
8. 按任意键关闭本次报警并返回正常显示。

## 运行现象

- 正常状态：OLED 显示 `Time:HH:MM:SS` 和 `Date:20YY-MM-DD`。
- 设置状态：OLED 显示当前设置项和数值，例如 `SET TIME HOUR`。
- 闹钟触发：OLED 显示 `*** ALARM ***` 和 `PRESS ANY KEY`，LED 与蜂鸣器交替提示。

## 按键功能

| 按键 | 功能 |
| --- | --- |
| KEY1 | 正常模式下进入设置；设置模式下切换下一项 |
| KEY2 | 当前数值加 1 |
| KEY3 | 当前数值减 1 |
| KEY4 | 保存当前时间、日期或闹钟设置 |

## 注意事项

- 保存时间和闹钟前，应避免按键连按导致设置项跳变。
- 如果 OLED 显示 `RTC INIT ERROR` 或提示检查 LSE，应检查 RTC 时钟源配置。
- 蜂鸣器为低电平触发，调试时注意不要长时间持续鸣叫。
