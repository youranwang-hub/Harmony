# DHT11 温湿度传感器拓展实验说明

本实验对应 `【外围实验1-1拓】DHT11传感器`，代码位于 `代码` 文件夹中。拓展版在基础温湿度读取上增加多次采样统计、最大值/最小值、成功/失败计数和超限报警。

## 实验目的

- 在 DHT11 基础读取功能上增加连续采样统计。
- 掌握最大值、最小值、成功次数和失败次数的记录方法。
- 根据温度和湿度阈值实现声光报警。
- 通过串口输出更完整的环境监测信息。

## 硬件连接

| 模块 | 引脚/资源 | 说明 |
| --- | --- | --- |
| DHT11 DATA | PE6 | 温湿度数据输入 |
| USART2_TX/RX | PA2 / PA3 | 串口输出调试信息 |
| LED | PB5 | 报警提示 |
| BEEP | PA8 | 蜂鸣器，低电平响 |

## 代码结构

```text
代码
├─BSP
│  ├─dht11.c / dht11.h       # DHT11 驱动
│  └─usart.c / usart.h       # 串口输出
├─USER
│  └─main.c                  # 采样统计、阈值判断、报警控制
└─UTILS
   ├─delay.c / delay.h       # 延时函数
   └─utils.c / utils.h       # 工具文件
```

## 核心流程

1. 初始化 USART2、DHT11、LED 和蜂鸣器。
2. 设置采样次数为 10 次，采样间隔为 10000 ms。
3. 每次读取 DHT11 数据并进行校验。
4. 读取成功时输出当前温湿度，并更新温度/湿度最大值和最小值。
5. 读取失败时关闭报警并增加失败计数。
6. 当温度超过 30.0 ℃或湿度超过 80.0 %时打开 LED 和蜂鸣器。
7. 采样结束后输出总采样次数、成功次数、失败次数和最终统计结果。

## 运行现象

串口会显示每次采样编号、温度、湿度、最大值、最小值和报警状态。超过阈值时输出报警提示，同时 LED 和蜂鸣器动作。

## 串口示例

```text
DHT11 Temperature and Humidity System
Sample Count: 10
Temperature Alarm: 30.0 C
Humidity Alarm: 80.0 %
---------- Sample 1/10 ----------
Temperature: 26.0 C
Humidity   : 45.0 %
ALARM: OFF
Environment status is normal.
```

## 关键参数

| 参数 | 值 |
| --- | --- |
| 采样次数 | 10 |
| 采样间隔 | 10000 ms |
| 温度报警阈值 | 30.0 ℃ |
| 湿度报警阈值 | 80.0 % |
| 报警 LED | PB5 |
| 报警蜂鸣器 | PA8 |

## 注意事项

- 温湿度阈值是课堂实验设置，可根据实际环境调整。
- DHT11 本身刷新速度较低，不适合高频连续读取。
- 读取失败时程序会关闭报警，避免因无效数据导致误报警。
- 若要脱离电脑使用，可继续加入 OLED 显示模块。
