# DHT11 温湿度传感器基础实验说明

本实验对应 `【外围实验1-1】DHT11传感器`，代码位于 `代码` 文件夹中。程序通过 PE6 读取 DHT11 温湿度数据，并通过 USART2 输出到上位机。

## 实验目的

- 理解 DHT11 单总线通信时序。
- 掌握 MCU 发送开始信号、等待响应、读取 40 位数据的流程。
- 学会使用校验和判断 DHT11 数据是否有效。
- 通过 USART2 将温湿度结果输出到串口工具。

## 硬件连接

| 模块 | 引脚/资源 | 说明 |
| --- | --- | --- |
| DHT11 DATA | PE6 | 单总线数据引脚 |
| DHT11 VCC | 开发板电源 | 按模块要求接 3.3 V 或 5 V |
| DHT11 GND | GND | 与开发板共地 |
| USART2_TX | PA2 | 串口输出 |
| USART2_RX | PA3 | 串口接收/回显 |

串口参数：115200 bit/s，8 位数据位，1 位停止位，无校验，无硬件流控。

## 代码结构

```text
代码
├─BSP
│  ├─dht11.c / dht11.h       # DHT11 驱动
│  └─usart.c / usart.h       # USART2 和 printf 重定向
├─USER
│  └─main.c                  # 主程序：读取温湿度并输出
└─UTILS
   ├─delay.c / delay.h       # 微秒/毫秒延时
   └─utils.c / utils.h       # 工具文件
```

## 核心流程

1. 初始化 USART2 和 DHT11。
2. MCU 将 PE6 配置为输出，拉低约 18 ms 发送开始信号。
3. MCU 拉高并切换 PE6 为输入模式，等待 DHT11 响应。
4. 依次读取湿度整数、湿度小数、温度整数、温度小数、校验和。
5. 校验成功则通过串口输出温湿度。
6. 校验失败或读取超时则输出错误提示。
7. 延时约 10 s 后进行下一次读取。

## 运行现象

串口输出初始化成功信息，随后周期显示温湿度。读取失败时输出 `Read DHT11 ERROR!`。

## 串口示例

```text
DHT11 Init OK
HUMI:45.0_TEMP:26.0
```

## 注意事项

- DHT11 对时序敏感，`delay_us()` 精度会直接影响读取成功率。
- 两次读取之间应保留足够间隔，避免传感器未完成刷新。
- 若长期读取失败，检查 PE6 数据线、上拉电阻、VCC、GND 和供电稳定性。
- `printf` 需要通过 `usart.c` 中的 `fputc()` 重定向到 USART2。
