# 实验2-1 LED BEEP

本实验使用 STM32F4 标准外设库配置 GPIO 输出，控制三盏板载 LED 和一个蜂鸣器。

## 文件说明

| 文件 | 功能 |
| --- | --- |
| `main.c` | 基础 GPIO 输出实验，分别验证 LED 和蜂鸣器的高低电平控制 |
| `mainplus.c` | 拓展实验，封装 LED/蜂鸣器控制宏，并按固定节奏循环显示多种组合状态 |

## 硬件连接

| 外设 | 引脚 | 有效电平 |
| --- | --- | --- |
| LED1 | PB5 | 低电平点亮 |
| LED2 | PB0 | 低电平点亮 |
| LED3 | PB1 | 低电平点亮 |
| 蜂鸣器 | PA8 | 低电平响 |

## 程序逻辑

`main.c` 的主要流程：

1. 使能 GPIOB 时钟。
2. 将 PB5、PB0、PB1 配置为推挽输出。
3. 使用 `GPIO_ResetBits()` 点亮 LED，使用 `GPIO_SetBits()` 熄灭 LED。
4. 使能 GPIOA 时钟。
5. 将 PA8 配置为推挽输出。
6. 使用低电平启动蜂鸣器，高电平关闭蜂鸣器。

因为基础程序中的点亮、熄灭、响铃和关闭语句连续执行，实际肉眼观察时最终状态通常是 LED 熄灭、蜂鸣器关闭。

`mainplus.c` 的主要流程：

1. 使用 `LED1_ON()`、`LED1_OFF()`、`BEEP_ON()`、`BEEP_OFF()` 等宏简化控制。
2. 使用 SysTick 实现毫秒级延时。
3. 在 `while(1)` 中每隔 1 秒切换一种 LED 与蜂鸣器组合状态。

## 运行现象

运行 `mainplus.c` 时，可以观察到 LED1、LED2、LED3 以及蜂鸣器按 7 组状态循环变化：

- LED1 单独点亮，蜂鸣器关闭。
- LED2 单独点亮，蜂鸣器响。
- LED3 单独点亮，蜂鸣器关闭。
- LED1 和 LED3 点亮，蜂鸣器响。
- 三个 LED 全部点亮，蜂鸣器关闭。
- 三个 LED 全部熄灭，蜂鸣器响。
- 三个 LED 全部熄灭，蜂鸣器关闭。

## 使用方法

在已有 STM32F407 标准外设库工程中，将需要运行的源码文件作为工程入口 `main.c` 使用，编译通过后下载到开发板观察现象。
