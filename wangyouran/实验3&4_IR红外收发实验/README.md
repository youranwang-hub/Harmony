# 实验3&4 — IR 红外收发实验

## 实验概述

基于 STM32F4 实现 NEC 红外协议的编码发送与解码接收。实验分为两部分：
- **实验3（发送）**：使用 TIM10 PWM 输出 38kHz 载波，按 NEC 协议编码发送数据
- **实验4（接收）**：使用 TIM9 输入捕获解析红外接收头信号，解码并校验数据

## 硬件连接

| 功能 | 引脚 | 外设 | 说明 |
|------|------|------|------|
| 红外接收 | PE5 | TIM9 CH1 | 输入捕获，上升沿触发 |
| 红外发送 | PB8 | TIM10 CH1 | PWM 输出，38kHz 载波 |
| 调试串口 | PA2 / PA3 | USART2 | 115200 波特率，printf 输出 |

## NEC 协议简述

NEC 协议使用 38kHz 载波调制，一帧完整数据如下：

```
       引导码 (Leader Code)
  ┌─────────┐      ┌─────────┐
  │  9ms     │      │  4.5ms  │
  │  载波ON  │      │  载波OFF │
──┘         └──────┘         └─────── ... 数据 ...

   数据位 0: 560μs ON + 560μs OFF
   数据位 1: 560μs ON + 1680μs OFF
   停止位:   560μs ON

   重复码:   9ms ON + 2.25ms OFF + 560μs ON
```

一帧包含：**引导码 + 地址码(8bit) + 地址反码(8bit) + 键码(8bit) + 键码反码(8bit) + 停止位**，共 32 位数据。

数据位采用 **LSB（低位先发）**，地址反码和键码反码用于校验。

## 核心模块说明

### 1. 红外发送 (`ir_send.c`)

#### DWT 精确延时
利用 Cortex-M4 的 **DWT 周期计数器** 实现微秒级精确延时，替代不可靠的软件循环延时：

```c
void systick_delay_us(uint32_t nus) {
    uint32_t cycles = nus * (SystemCoreClock / 1000000); // 168MHz: 1μs = 168 cycles
    uint32_t start  = DWT->CYCCNT;
    while((DWT->CYCCNT - start) < cycles);
}
```

#### 38kHz PWM 载波生成
- 定时器：**TIM10 CH1**（APB2 总线，168MHz）
- `TIM_Prescaler = 0`（不分频）
- `TIM_Period = 4421` → 频率 = 168M / 4421 ≈ **38kHz**
- 占空比：`TIM_Pulse = 2210`，约 50%
- PWM 模式 1，极性 High

#### 载波开关控制
通过 `TIM_ForcedOC1Config` 在 PWM1 和强制无效电平之间切换，实现载波的启停：
- **PwmEnable()**：恢复 PWM1 模式，输出 38kHz 载波
- **PwmDisable()**：强制输出低电平，关闭载波

#### NEC 编码流程 (`IR_Send`)
```
引导头 → 地址 → ~地址 → 键值 → ~键值 → 停止位
  9ms     8bit    8bit    8bit    8bit    560μs
 4.5ms
```

### 2. 红外接收 (`ir_recv.c`)

#### 输入捕获配置
- 定时器：**TIM9 CH1**（APB2 总线）
- 预分频：`TIM_Prescaler = 167` → 计数频率 1MHz（1μs 分辨率）
- 周期：`TIM_Period = 10000`（10ms 溢出，用于帧超时检测）
- 捕获：上升沿触发，直连模式，8 周期滤波防毛刺

#### 边沿检测原理
红外接收头（如 HS0038）收到载波时输出低电平，无载波时输出高电平。捕获相邻两个**上升沿**的时间差，减去固定的 560μs 低脉冲宽度，即可得到高电平持续时间：

| 高电平时长 | 判断 |
|-----------|------|
| ~560μs | 数据位 0 |
| ~1680μs | 数据位 1 |
| ~2250μs | 重复码 |
| ~4500μs | 引导码 |

#### 位计数器防移位
引入 `IRBitCount` 变量追踪已接收位数（0~32），收满 32 位后忽略 stop bit 造成的额外上升沿，**防止数据被误移位**。收满后立即在 CC1 中断中设置 `DATA_OK` 标志，无需等待定时器溢出。

#### 溢出中断
仅用于异常恢复：若收到引导码但未能在 10ms 内收满 32 位，清除状态标志进入等待。

#### 数据校验 (`IR_Recv`)
主循环轮询处理，从 32 位 IRData 中提取 4 个字节：
1. 键码反码校验：`byte3 == ~byte4`
2. 地址格式判断：若 `byte1 != ~byte2` 则为扩展协议（16 位地址），否则为8位地址

#### 弱函数回调
```c
__attribute__((weak)) void IR_Rece_Proc(uint16_t addr, uint8_t code);
```
默认实现通过 printf 输出地址和键码，应用程序可重写实现自定义处理。

### 3. 主程序 (`main.c`)

#### 时序架构
- **SysTick**：1ms 中断，驱动全局毫秒计数器 `systick_ms`
- **非阻塞轮询**：IR_Recv() 每次循环调用，不阻塞
- **定时发送**：`systick_ms` 差值比较，每 10 秒发送一次
- **自干扰防护**：`ir_sending` 标志，发送期间暂停接收

#### 主循环流程
```
while(1):
  ├── if (!ir_sending): IR_Recv()          # 非发送时轮询接收
  └── if (已过10秒): IR_Send(1, 123)       # 定时发送，带自干扰防护
      ├── ir_sending = 1
      ├── IR_Send(1, 123)                  # 阻塞约 67.5ms
      └── ir_sending = 0
```

### 4. USART2 调试输出 (`usart.c`)

- 重定向 `fputc()` → printf 输出到 USART2
- 重定向 `fgetc()` → scanf/getchar 从 USART2 读取
- 回显中断：收到数据后立即回发，用于串口助手调试

## 运行效果

连接串口助手（115200, 8N1），启动后输出：

```
IR Recv Init OK
IR Send Init OK
```

- 每 10 秒自动发送：`IRSend:addr:1,code:123`
- 收到遥控器信号时打印：`IRRecv:addr:1, code:69`

## 关键技术点

| 技术 | 说明 |
|------|------|
| NEC 协议 | 38kHz 载波调制，LSB 先发，引导码+32位数据帧 |
| TIM 输出比较 | TIM10 PWM1 模式生成 38kHz 方波载波 |
| TIM 输入捕获 | TIM9 上升沿捕获 + 时差测量解析信号定时 |
| DWT 周期计数 | CPU 时钟级精确微秒延时，替代不可靠的软件循环 |
| SysTick 定时 | 1ms 系统节拍，驱动主循环非阻塞定时 |
| 弱函数 | `__attribute__((weak))` 支持应用层自定义回调 |
| 位计数器 | 防 stop bit 造成数据移位，严格收 32 位即停 |
| 自干扰防护 | 发送时屏蔽接收，避免收到自己发出的红外信号 |
