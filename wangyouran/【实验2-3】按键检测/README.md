# 实验 2-3：GPIO 输入 — 按键读取与抬起检测

## 实验目的

掌握 GPIO 输入模式的使用，通过读取按键电平实现 LED 控制，并进一步实现按键抬起检测功能。

## 硬件连接

| 外设  | GPIO 引脚 | 说明                       |
|-------|-----------|----------------------------|
| KEY1  | PA0       | 上拉输入，按下 = 低电平(0)   |
| KEY2  | PC13      | 上拉输入，按下 = 低电平(0)   |
| KEY3  | PC6       | 上拉输入，按下 = 低电平(0)   |
| KEY4  | PC7       | 上拉输入，按下 = 低电平(0)   |
| LED1  | PB5       | 推挽输出，低电平点亮          |
| LED2  | PB0       | 推挽输出，低电平点亮          |
| LED3  | PB1       | 推挽输出，低电平点亮          |

## 按键消抖

按键采用软件消抖：首次检测到按下后延时 20ms，再次读取确认，避免机械抖动导致的误触发。

## 版本说明

### 基础版 (`basic/main.c`)

实现按键按下即响应的基本功能：

| 按键   | 按下时效果 |
|--------|-----------|
| KEY1   | LED1 亮   |
| KEY2   | LED2 亮   |
| KEY3   | LED3 亮   |
| KEY4   | 全部 LED 熄灭 |

### 拓展版 (`extended/main.c`)

在基础版之上新增**按键抬起检测**功能：

- 检测到按键后不立即响应，而是进入 `Wait_KeyRelease()` 函数原地等待
- 持续读取引脚直到按键松开（恢复高电平），然后才执行操作
- **翻转模式**：LED1~3 改用 `GPIO_ToggleBits`，实现按一次亮、再按一次灭

| 按键   | 操作                    |
|--------|------------------------|
| KEY1   | 按下→抬起 → **翻转** LED1 |
| KEY2   | 按下→抬起 → **翻转** LED2 |
| KEY3   | 按下→抬起 → **翻转** LED3 |
| KEY4   | 按下→抬起 → 全部熄灭      |

**`Wait_KeyRelease()` 原理**：

```c
void Wait_KeyRelease(GPIO_TypeDef* GPIOx, uint16_t GPIO_Pin)
{
    while (GPIO_ReadInputDataBit(GPIOx, GPIO_Pin) == 0)
    {
        /* 原地等待，直到松开 */
    }
}
```

## 两版对比

| 项目       | 基础版       | 拓展版               |
|------------|-------------|---------------------|
| 响应时机    | 按下即响应    | 抬起后才响应           |
| LED 控制    | 按下亮/全灭  | 翻转（toggle）/ 全灭   |
| 抬起检测    | 无           | 有                   |
