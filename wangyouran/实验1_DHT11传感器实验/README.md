# 实验 1：DHT11 温湿度传感器实验

## 实验目的

掌握 DHT11 单总线通信协议，学习 GPIO 输入输出模式动态切换，实现温湿度数据的采集与串口打印。

## 硬件连接

| 外设 | 引脚 | 说明 |
|------|------|------|
| DHT11 DATA | PE6 | 单总线数据线，需外接 4.7kΩ 上拉电阻 |
| 串口 TX | PA2 | 连接 USB 转 TTL 模块 RXD |
| 串口 RX | PA3 | 连接 USB 转 TTL 模块 TXD |

## 串口配置参数（USART2）

| 参数 | 值 | 说明 |
|------|-----|------|
| 波特率 | 115200 | |
| 数据位 | 8bit | |
| 停止位 | 1bit | |
| 校验位 | 无 | |
| 硬件流控 | 无 | |
| printf 重定向 | `fputc` → USART2 | |

## DHT11 传感器简介

DHT11 是一款数字温湿度传感器，采用单总线协议（一根数据线完成双向通信）。

| 参数 | 范围 | 精度 |
|------|------|------|
| 温度 | 0 ~ 50℃ | ±2℃ |
| 湿度 | 20 ~ 90%RH | ±5%RH |
| 采样周期 | ≥ 1s | |

## 单总线通信协议

### 数据格式（40 bit，高位先出）

| 字节 | 内容 | 说明 |
|------|------|------|
| Byte 1 | 湿度整数部分 | 如 45 → 45%RH |
| Byte 2 | 湿度小数部分 | DHT11 固定为 0 |
| Byte 3 | 温度整数部分 | 如 25 → 25℃ |
| Byte 4 | 温度小数部分 | DHT11 固定为 0 |
| Byte 5 | 校验和 | = 前 4 字节之和（低 8 位） |

### 时序流程

```
MCU                          DHT11
 │                              │
 │── 拉低 18ms ────────────────→│  起始信号
 │── 拉高 30μs ────────────────→│  
 │                              │
 │←── 拉低 80μs ────────────────│  响应信号
 │←── 拉高 80μs ────────────────│  
 │                              │
 │←── 数据 bit 0 (50μs) ───────│  40 bit 数据
 │←── 数据 bit 1 (70μs) ───────│  
 │     ...                      │
 │←── 拉低 50μs ────────────────│  结束信号
```

### 0 和 1 的区分

| bit 值 | 低电平 | 高电平 |
|--------|--------|--------|
| **0** | 50μs | 26~28μs |
| **1** | 50μs | 70μs |

读取策略：在低电平结束 + 40μs 后采样，仍为高 = 1，已变低 = 0。

## 数据结构（dht11.h）

```c
typedef struct {
    uint8_t humi_int;   // 湿度整数
    uint8_t humi_deci;  // 湿度小数
    uint8_t temp_int;   // 温度整数
    uint8_t temp_deci;  // 温度小数
    uint8_t check_sum;  // 校验和
} DHT11_Data;
```

## 核心函数

### 模块内部函数（static）

| 函数 | 功能 |
|------|------|
| `DHT11_Mode_IN()` | 将 PE6 切换为输入模式 |
| `DHT11_Mode_OUT()` | 将 PE6 切换为推挽输出模式 |
| `DHT11_ReadBit()` | 读取 1 个数据位，返回 0/1/-1（超时） |
| `DHT11_ReadByte()` | 读取 1 字节（调用 8 次 ReadBit，MSB 先行） |
| `DHT11_WaitFor()` | 带超时的电平等待，防止死循环 |

### 公开函数

| 函数 | 功能 | 返回值 |
|------|------|--------|
| `DHT11_Init()` | 初始化 GPIOE6，时钟使能，默认拉高 | — |
| `DHT11_ReadData(pData)` | 发送起始信号并读取 40bit 数据 | 0 成功 / -1 失败 |

## 运行流程

```
上电/复位
  ↓
初始化 USART2（115200 8N1）
  ↓
初始化 DHT11（PE6 输出模式，初始拉高）
  ↓
printf "DHT11 Init OK"
  ↓
delay 1s
  ↓
┌─ while(1) ──────────────────────────────────┐
│                                              │
│  发送起始信号 → 等响应 → 读 5 字节            │
│     ↓                                        │
│  校验通过？                                  │
│   ├─ 是 → printf "HUMI:xx.x_TEMP:xx.x"       │
│   └─ 否 → printf "Read DHT11 ERROR!"         │
│     ↓                                        │
│  delay 10s → 循环                             │
└──────────────────────────────────────────────┘
```

## 预期现象

打开串口助手（115200 8N1），上电后显示：

```
DHT11 Init OK
HUMI:45.0_TEMP:25.0
HUMI:45.0_TEMP:25.0
...
```

用手触摸 DHT11 传感器，温度读数会上涨。

## 关键技术点

### GPIO 模式动态切换

单总线的核心技巧是同一引脚在**输出**和**输入**之间来回切换：

```
输出模式 → 发起始信号（拉低 18ms + 拉高 30μs）
输入模式 → 等待响应 + 读取数据
```

### 延时精度要求

DHT11 的 bit 区分依赖于 μs 级时序（0 的高电平约 26μs，1 约 70μs）。代码使用软件循环延时（`delay_us`），需注意编译器优化等级对延时精度的影响。

### 校验机制

5 字节数据中，第 5 字节为前 4 字节之和的低 8 位。校验失败说明通信过程中有干扰或时序不对。
