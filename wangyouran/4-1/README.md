# 实验 4-1：串口指令控制 LED 与蜂鸣器

## 实验目的

通过 USART2 串口接收用户指令，实现对 LED 和蜂鸣器的远程控制。掌握 STM32 的 USART 配置、中断接收、帧解析以及串口指令响应机制。

## 硬件连接

| 外设 | 引脚 | 说明 |
|------|------|------|
| LED1 | PB5 | 低电平点亮 |
| LED2 | PB0 | 低电平点亮 |
| LED3 | PB1 | 低电平点亮 |
| 蜂鸣器 | PA8 | 低电平发声 |
| USART2 TX | PA2 | 串口发送 |
| USART2 RX | PA3 | 串口接收 |

## 串口配置

| 参数 | 值 |
|------|------|
| 波特率 | 115200 |
| 数据位 | 8 |
| 停止位 | 1 |
| 校验位 | 无 |
| 帧结束符 | `\r\n`（回车换行） |
| 中断分组 | Group 1（1 位抢占 + 3 位子优先级） |

## 基础版 (basic/main.c)

### 功能

通过串口发送 4 字节指令，点亮指定 LED（200ms 后自动熄灭）或触发蜂鸣器。

### 支持的指令

| 指令 | 功能 | 串口回复 | 备注 |
|------|------|----------|------|
| `LED1` | 点亮 LED1 | `LED1` | 200ms 后自动熄灭 |
| `LED2` | 点亮 LED2 | `LED2` | 200ms 后自动熄灭 |
| `LED3` | 点亮 LED3 | `LED3` | 200ms 后自动熄灭 |
| `BEEP` | 蜂鸣器响一声 | `蜂鸣器` | 50ms 后停止 |
| 其他 | — | `指令错误！` | |

### 代码修复记录

基础版初始代码存在以下问题，已逐一修复：

| # | 问题 | 修复方式 |
|---|------|----------|
| 1 | `delay_us`/`delay_ms` 重复定义导致编译失败 | 删除重复代码 |
| 2 | 使用 `USART_FLAG_TC`，首次发送可能卡死 | 改为 `USART_FLAG_TXE` |
| 3 | 接收缓存 `Rx_Con` 无越界保护 | 添加 `if(Rx_Con < 63)` 判断 |
| 4 | `PriorityGroup_0` 不支持抢占优先级 1 | 改为 `PriorityGroup_1` |

## 拓展版 (extended/main.c)

### 功能

在基础版之上重构，改为**独立开关控制**，LED 点亮后会一直保持，直到收到熄灭指令。支持单独控制每个 LED 和全亮/全灭批量操作。

### 支持的指令

| 指令 | 功能 | 串口回复 | 说明 |
|------|------|----------|------|
| `L1++` | LED1 点亮 | `LED1 ON` | 保持亮，不会自动灭 |
| `L1--` | LED1 熄灭 | `LED1 OFF` | |
| `L2++` | LED2 点亮 | `LED2 ON` | 保持亮 |
| `L2--` | LED2 熄灭 | `LED2 OFF` | |
| `L3++` | LED3 点亮 | `LED3 ON` | 保持亮 |
| `L3--` | LED3 熄灭 | `LED3 OFF` | |
| `AL++` | 三个 LED 全亮 | `ALL ON` | 批量操作 |
| `AL--` | 三个 LED 全灭 | `ALL OFF` | 批量操作 |
| `BEEP` | 蜂鸣器响一声 | `BEEP OK` | 80ms 后停止 |
| 其他 | — | `指令错误！` | |

### 指令格式

所有指令均为 4 个字符（不含回车），格式为 `动作+目标+操作+操作`：

- **单独控制**：`L` + `{1/2/3}` + `{+/-}` + `{+/-}`（如 `L1++`、`L1--`，第 4 字节与第 3 字节保持一致即可）
- **批量控制**：`A` + `L` + `{+/+/ -/-}`（如 `AL++`、`AL--`）
- **蜂鸣器**：`B` + `E` + `E` + `P`

## 基础版 vs 拓展版对比

| 特性 | 基础版 | 拓展版 |
|------|--------|--------|
| LED 状态 | 200ms 后自动熄灭 | 保持，直到手动关闭 |
| 熄灭控制 | 不支持 | 独立 `Lx-` 指令 |
| 批量操作 | 不支持 | `AL++` / `AL--` |
| 指令格式 | `LED1` | `L1++` / `L1--` |
| 蜂鸣器时长 | 50ms | 80ms |
| 串口反馈 | 中文/简短 | 英文/详细状态 |

## 使用方式

1. 用 Keil MDK 打开工程，添加对应 `main.c` 并编译烧录
2. 打开串口调试助手，波特率 115200，8N1
3. 发送指令末尾需带回车换行（`\r\n`）
4. 上电后拓展版会自动打印指令列表
