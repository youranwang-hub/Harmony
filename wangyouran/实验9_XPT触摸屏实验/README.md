# 实验9 — XPT 触摸屏实验

## 实验目的

- 理解触摸屏工作原理
- 掌握 XPT2046 芯片原理与交互方法
- 掌握 SPI 总线模拟实现方法
- 掌握电阻触摸屏 ADC 数据转换的控制方法
- 掌握 XPT2046 坐标转换方式

## 实验概述

触摸屏是当前最常用的输入部件，常与 LCD 模块集成一起使用。AU100（青软遨游 100）的 LCD 液晶屏上集成了 **XPT2046** 触摸屏控制芯片，内置 ADC，并带有电阻屏。本实验通过此触摸屏实现 LCD 触摸点坐标的获取和转换。

通过本实验可深入了解触摸屏的工作原理，加深掌握 SPI 编程方法，掌握电阻触摸屏 ADC 数据读取、滤波及校正算法。实验内容主要如下：

1. 触摸屏工程创建
2. 触摸屏基础交互实现
3. 触摸屏坐标读取实现
4. 触摸屏测试示例实现

XPT2046 是 4 线制电阻触摸屏控制芯片，通过 **SPI 接口** 与 MCU 通讯，内置 **12-bit ADC**（0~4095）。MCU 通过 SPI 写入 8-bit 控制字选择通道并启动 ADC，随后在 SPI 时钟下连续读取 12-bit 转换结果，即可得到触摸点 X/Y 方向的电压值。

## 硬件连接

| 功能 | 引脚 | 方向 | 说明 |
|------|------|------|------|
| SPI SCK | PE0 | GPIO 推挽输出 | SPI 时钟（软件模拟） |
| SPI MOSI (DIN) | PE2 | GPIO 推挽输出 | 主机 → XPT2046（控制字） |
| SPI MISO (DOUT) | PE3 | GPIO 上拉输入 | XPT2046 → 主机（ADC 数据） |
| SPI CS | PD13 | GPIO 推挽输出 | 片选，低电平有效 |
| PENIRQ | PE4 | GPIO 上拉输入 | 触摸中断请求，**低电平**表示有触摸 |
| 调试串口 | PA2 / PA3 | USART2 | 115200 波特率，printf 输出 |

> XPT2046 与 LCD 共用 SPI 总线但不与 LCD 的 FSMC 复用——本实验**未使用** STM32 硬件 SPI，全部通过 GPIO **软件模拟 SPI 时序**，便于理解 SPI 协议细节和跨平台移植。

## 触摸屏工作原理

### 1. 电阻触摸屏结构

4 线制电阻触摸屏由两层透明导电层（ITO，氧化铟锡）构成：

```
        上层 ITO (X 方向)
  ┌─────────────────────────────┐
  │   X+ ────────────── X-      │  ← 一对横向电极
  │                             │
  ├─────────────────────────────┤
  │                             │  ← 隔离点（spacer dot）
  │   按压时两层导通，形成分压      │
  ├─────────────────────────────┤
  │   Y+ ────────────── Y-      │  ← 一对纵向电极
  │                             │
  └─────────────────────────────┘
        下层 ITO (Y 方向)
```

### 2. 触摸坐标测量原理

电阻屏本质是**二维电压分压器**，通过切换电极方向分别测量 X/Y 坐标：

#### 测量 X 坐标（水平位置）

```
       VREF (VDD)
        │
       Y+ ──────┐
                │
   ┌────────┐   │   ┌──────┐
   │        │   │   │      │
   │  上层  │───●───│ 下层 │  ← 按压点（短路点）
   │  ITO   │   │   │ ITO  │
   │        │   │   │      │
   └────────┘   │   └──────┘
                │
       Y- ──────┘
                │
               GND

  X+ ──→ A/D 测量 X+ 端电压
```

**步骤**：
1. Y+ 接 VREF，Y- 接 GND
2. 按压点将上下层短路，形成水平方向分压
3. X+ 端测得的电压 V = VREF × (Rx1 / (Rx1+Rx2))
4. 通过 X+ 电压推算 X 位置

#### 测量 Y 坐标（垂直位置）

对称地切换电极：X+ 接 VREF，X- 接 GND，Y+ 测电压。

**核心要点**：
- 切换电极方向由 XPT2046 内部模拟开关（FET Switch）完成
- 主机通过 SPI 写入不同**通道控制字**（0x90 / 0xD0）切换测量方向
- 测量过程**不影响触摸状态**

### 3. XPT2046 控制器

| 参数 | 值 |
|------|-----|
| ADC 分辨率 | 12-bit（0~4095） |
| 模拟通道数 | 8 路（AIN0~AIN7），含温度、电池检测 |
| 通讯接口 | SPI 4 线（CS / DCLK / DIN / DOUT），最高 2.5 MHz |
| 内置参考电压 | 2.5V |
| 触摸中断 | PENIRQ 引脚，触摸时拉低 |
| 测量对象 | X+ / Y+ 位置、温度 T0/T1、电池电压 VBAT、辅助输入 AUX |
| 工作电压 | 2.2V ~ 3.6V |
| 封装 | TSSOP-16 / QFN-16 |

**模拟输入通道分配**：

| 控制字高 3 位 | 通道 | 说明 |
|------|------|------|
| `001` | X+ | 测量 X 坐标（Y+/Y- 加电） |
| `101` | Y+ | 测量 Y 坐标（X+/X- 加电） |
| `000` | TEMP0 | 内部温度传感器 0 |
| `010` | VBAT | 电池电压检测 |
| `110` | AUX | 辅助模拟输入 |
| `011` | TEMP1 | 温度传感器 1 |

## SPI 模拟实现

### 1. XPT2046 SPI 时序要求

```
        ┌─────┐                                       ┌──────┐
CS  ────┘     └─────────────────────────────────────────┘
                      ┌──┐  ┌──┐  ┌──┐  ┌──┐  ┌──┐
DCLK  ─────────────────┘  └──┘  └──┘  └──┘  └──┘  └─────────
        ┌───┐ ┌───┐ ┌───┐ ┌───┐ ┌───┐ ┌───┐ ┌───┐ ┌───┐
DIN   ───┤D7 ├─┤D6 ├─┤D5 ├─┤D4 ├─┤D3 ├─┤D2 ├─┤D1 ├─┤D0 ├───
        └───┘ └───┘ └───┘ └───┘ └───┘ └───┘ └───┘ └───┘
        ←───── 8 bit 控制字（写） ────→

        ←─────── DCLK 继续 12 个周期 ─────────→
        ──────────  DOUT 输出 12 bit ADC 数据 ────
                ┌──────────┐ ... ┌──────────┐
DOUT  ─────┐───┘          └─────┘           └───
          [D11]                              [D0]
```

**关键时序要求**：
- **DCLK 上升沿**：DIN 数据被锁存
- **DCLK 下降沿**：DOUT 数据被采样
- DIN（控制字）共 8 bit，前 8 个 DCLK 周期
- DOUT（ADC 数据）共 12 bit，后 12 个 DCLK 周期
- 总计 **20 个 DCLK 周期**完成一次完整通讯

### 2. SPI 写命令（8 bit）

```c
static void XPT2046_WriteCMD(uint8_t ucCmd) {
    uint8_t i;
    XPT2046_MOSI_0();
    XPT2046_CLK_LOW();

    for (i = 0; i < 8; i++) {
        // 1. MOSI 输出当前位 (MSB 先发)
        ((ucCmd >> (7 - i)) & 0x01) ? XPT2046_MOSI_1() : XPT2046_MOSI_0();
        XPT2046_DelayUS(5);
        // 2. CLK 拉高 → 从机在上升沿采样
        XPT2046_CLK_HIGH();
        XPT2046_DelayUS(5);
        // 3. CLK 拉低 → 完成一位
        XPT2046_CLK_LOW();
    }
}
```

### 3. SPI 读命令（12 bit）

```c
static uint16_t XPT2046_ReadCMD(void) {
    uint8_t i;
    uint16_t usBuf = 0, usTemp;

    XPT2046_MOSI_0();
    XPT2046_CLK_HIGH();

    for (i = 0; i < 12; i++) {
        XPT2046_CLK_LOW();                    // 下降沿：从机输出下一位
        usTemp = XPT2046_MISO();              // 读取 MISO
        usBuf |= usTemp << (11 - i);          // 组合 12 bit 数据
        XPT2046_CLK_HIGH();                   // 上升沿
    }
    return usBuf;
}
```

### 4. ADC 采样整合

```c
static uint16_t XPT2046_ReadAdc(uint8_t ucChannel) {
    XPT2046_WriteCMD(ucChannel);   // 写控制字（选通道 + 启动 ADC）
    return XPT2046_ReadCMD();       // 读 12 bit ADC 数据
}
```

**通道控制字**（本实验使用）：

| 宏 | 值 | 二进制 | 含义 |
|----|-----|--------|------|
| `XPT2046_CHANNEL_X` | 0x90 | 1001 0000 | 测量 X 坐标（AIN=001，差分模式，掉电=00） |
| `XPT2046_CHANNEL_Y` | 0xD0 | 1101 0000 | 测量 Y 坐标（AIN=101，差分模式，掉电=00） |

**控制字位定义**：

| 位 | 名称 | 含义 |
|----|------|------|
| 7 | S | Start，固定 1 |
| 6-4 | A2-A0 | 通道选择（000~111） |
| 3 | MODE | 1=8bit, 0=12bit（**本实验用 12bit**） |
| 2 | SER/DFR | 1=单端, 0=差分（**本实验用差分**） |
| 1-0 | PD1-PD0 | 00=始终上电, 01=参考关断, 10=ADC 关断, 11=全关 |

## 核心模块说明

### 1. XPT2046 初始化 (`XPT2046_Init`)

```c
void XPT2046_Init(void) {
    RCC_AHB1PeriphClockCmd(RCC_AHB1Periph_GPIOD | RCC_AHB1Periph_GPIOE, ENABLE);

    // SCK (PE0) / MOSI (PE2) / CS (PD13) - 推挽输出, 25MHz
    // MISO (PE3) - 上拉输入
    // PENIRQ (PE4) - 上拉输入

    XPT2046_CS_ENABLE();   // 拉低片选
}
```

**PENIRQ 引脚初始电平为高**（无触摸），按下触摸时 XPT2046 拉低 PENIRQ 触发外部中断或状态机轮询。

### 2. ADC 双通道读取

```c
static void XPT2046_ReadAdcXY(uint16_t *sX_Ad, uint16_t *sY_Ad) {
    *sX_Ad = XPT2046_ReadAdc(XPT2046_CHANNEL_X);   // 读 X 坐标 ADC
    XPT2046_DelayUS(1);                              // 通道切换稳定
    *sY_Ad = XPT2046_ReadAdc(XPT2046_CHANNEL_Y);   // 读 Y 坐标 ADC
}
```

由于一次完整读操作需要 20 个 DCLK 周期（约 8.3μs @ 2.4MHz），加上通道切换稳定延时，单次双通道读取约 **20~50 μs**。

### 3. 滤波算法 (`XPT2046_FilterXY`)

电阻触摸屏存在电磁噪声、机械抖动、ADC 量化误差等干扰，**单次采样不可信**。本实验采用**去极值平均滤波**（中位值平均滤波）：

```c
#define SAMPLE_COUNT  10   // 采样次数

static int8_t XPT2046_FilterXY(XPT2046_Coord *pCoord) {
    uint16_t sBuffer[2][10] = {0};          // X/Y 各采样 10 次
    uint16_t lMin, lMax, lSum;

    // 1. 连续采样 10 次（中途抬笔则中止）
    do {
        XPT2046_ReadAdcXY(&sBuffer[0][i], &sBuffer[1][i]);
    } while ((XPT2046_ReadIRQ() == XPT2046_PENIRQ_Active) && (i < 10));

    if (i < 10) return -1;   // 采样中途抬手，本次无效

    // 2. 找出 X/Y 的最大最小值，并累加
    for (i = 0; i < 10; i++) {
        if (sBuffer[0][i] < xMin) xMin = sBuffer[0][i];
        if (sBuffer[0][i] > xMax) xMax = sBuffer[0][i];
        xSum += sBuffer[0][i];
        // ... Y 通道同理
    }

    // 3. 去掉 1 个最大值 + 1 个最小值，对剩余 8 个求平均
    pCoord->x = (xSum - xMin - xMax) >> 3;   // ÷ 8
    pCoord->y = (ySum - yMin - yMax) >> 3;   // ÷ 8
    return 0;
}
```

**算法优点**：
- 有效剔除**突发毛刺**（如电磁干扰、机械抖动）
- 相比简单平均，**对极端值不敏感**
- 10 次采样约需 0.5ms，对实时性影响可忽略

**算法局限**：
- 若噪声是高斯白噪声，效率与简单平均接近
- 对周期性干扰（如 50Hz 工频）效果有限——可加滑动平均或卡尔曼滤波

### 4. 坐标仿射变换 (`XPT2046_GetTouchPoint`)

电阻触摸屏的 ADC 值（0~4095）需要映射到 LCD 像素坐标（0~239 / 0~319）。由于触摸屏的安装位置、旋转角度、缩放系数各不相同，**简单线性映射不准确**。

**采用 6 参数仿射变换**：

```
  XL = A·X + B·Y + C
  YL = D·X + E·Y + F
```

其中：
- `(X, Y)` 是 XPT2046 的原始 ADC 值
- `(XL, YL)` 是 LCD 像素坐标
- `A, B, C, D, E, F` 是 6 个校准系数

```c
int8_t XPT2046_GetTouchPoint(XPT2046_Coord *pLCD) {
    XPT2046_Coord adcCoord;

    if (XPT2046_FilterXY(&adcCoord) < 0) return -1;

    // 仿射变换应用
    pLCD->x = TouchFactor[LCD_SCAN_MODE].A * adcCoord.x +
              TouchFactor[LCD_SCAN_MODE].B * adcCoord.y +
              TouchFactor[LCD_SCAN_MODE].C;

    pLCD->y = TouchFactor[LCD_SCAN_MODE].D * adcCoord.x +
              TouchFactor[LCD_SCAN_MODE].E * adcCoord.y +
              TouchFactor[LCD_SCAN_MODE].F;
    return 0;
}
```

**为什么是仿射变换？** 仿射变换包含**旋转、缩放、平移、错切**（6 自由度），可描述触摸屏与 LCD 之间的任何线性映射关系。通过 3 个已知校准点（3 对坐标）即可解出 6 个参数。

**校准方法**：
- 在 LCD 上画 3 个已知像素坐标 `(XL1, YL1) (XL2, YL2) (XL3, YL3)`
- 依次触摸这 3 个点，记录 ADC 坐标 `(X1, Y1) (X2, Y2) (X3, Y3)`
- 联立方程：
  ```
  | X1 Y1 1  0  0  0 |   |A|   |XL1|
  | 0  0  0  X1 Y1 1 | × |B| = |YL1|
  | X2 Y2 1  0  0  0 |   |C|   |XL2|
  | 0  0  0  X2 Y2 1 |   |D| = |YL2|
  | X3 Y3 1  0  0  0 |   |E|   |XL3|
  | 0  0  0  X3 Y3 1 |   |F|   |YL3|
  ```
- 求解 6 元一次方程组即得校准系数

### 5. 8 套校准参数

由于 LCD 扫描方向（0~7）不同，触摸屏与 LCD 的几何关系也不同，因此为**每种扫描方向预设了不同的校准参数**：

```c

     0.002782,  0.061522, -11.595689,  0.083393,  0.005159, -15.650089,  // 扫描 2
     0.089743, -0.000289, -20.612209, -0.001374,  0.064451, -16.054003,  // 扫描 3
     0.000767, -0.068258,  250.891769, -0.085559, -0.000195, 334.747650,  // 扫描 4
    -0.084744,  0.000047,  323.163147, -0.002109, -0.066371, 260.985809,  // 扫描 5
    -0.001848,  0.066984, -12.807136, -0.084858, -0.000805, 333.395386,  // 扫描 6
```

### 6. 触摸状态机 (`XPT2046_DetectTouch`)

本实验采用**三态有限状态机**（FSM）实现触摸检测和消抖：

```
     第一次低电平                 连续低电平 ≥ DURIATION_TIME
┌──────────┐  ┌──────────┐  DURIATION_TIME  ┌──────────┐
│  RELEASE │──│ WAITING  │─────────────────→│  PRESSED │
│  (释放)  │  │  (等待消抖)  │                  │  (按下)  │
└──────────┘  └────┬─────┘                  └────┬─────┘
     ↑            │ 高电平（抖动）                  │ 高电平（抬手）
     └────────────┴─────────────────────────────────┘
                       转回 RELEASE
```

**状态转换表**：

| 当前状态 | 事件（IRQ 引脚） | 动作 | 下一状态 |
|----------|-----------------|------|----------|
| RELEASE | 低电平 | 记录首次按下 | WAITING |
| WAITING | 低电平，累加次数 | 若 ≥ DURIATION_TIME 则确认按下 | PRESSED |
| WAITING | 高电平 | 抖动，重置计数器 | RELEASE |
| PRESSED | 低电平 | 持续按下 | PRESSED |
| PRESSED | 高电平 | 抬手 | RELEASE |

**消抖时间**：

```c
#define DURIATION_TIME  2    // 2 次计数

// 消抖时间 = DURIATION_TIME × 函数调用间隔
// 本实验主循环每 50ms 调用一次 → 消抖时间 = 2 × 50ms = 100ms
```

**回调函数**（弱函数，支持应用层重写）：

```c
__weak void XPT2046_TouchDown(XPT2046_Coord *touch) {
    printf("TouchDown,x:%d,y:%d\r\n", touch->x, touch->y);
}

__weak void XPT2046_TouchUp(XPT2046_Coord *touch) {
    printf("TouchUp,x:%d,y:%d\r\n", touch->x, touch->y);
}
```

应用层可在其他 `.c` 文件中**重新定义同名非 weak 函数**覆盖默认实现，从而实现自定义触摸处理（如画点、触发按钮等）。

## 关键技术点

| 技术 | 说明 |
|------|------|
| 电阻触摸屏 | 两层 ITO + 隔离点，X/Y 分压测量原理 |
| 4 线制 | X+ / X- / Y+ / Y- 四电极分时切换 |
| XPT2046 | 12-bit ADC，SPI 接口，8 通道模拟开关 |
| 软件 SPI | GPIO 模拟 SCK / MOSI / MISO / CS，跨平台可移植 |
| 8-bit 控制字 | S+A2-A0+MODE+SER/DFR+PD1-PD0 选择通道和工作模式 |
| 12-bit ADC | 单次转换 20 个 DCLK 周期，DOUT 紧随 DIN 之后 |
| 去极值平均滤波 | 10 次采样去最大最小后求平均，剔除毛刺 |
| 仿射变换 | 6 参数矩阵描述旋转/缩放/平移/错切几何变换 |
| 3 点校准 | 3 对坐标联立方程组，解 6 元一次方程 |
| 多方向参数 | 8 套校准参数匹配 8 种 LCD 扫描方向 |
| 状态机消抖 | 三态 FSM（RELEASE / WAITING / PRESSED） |
| PENIRQ 中断源 | XPT2046 拉低表示触摸，**未接外部中断**，本实验采用轮询 |
| 弱函数回调 | `__weak` 允许应用层重写 TouchDown / TouchUp |
| printf 重定向 | `fputc` 映射到 USART2，方便调试坐标输出 |

## 代码注意事项

1. **PENIRQ 引脚电平定义**：`XPT2046_PENIRQ_Active = 0` 表示**低电平有效**——XPT2046 在有触摸时拉低 PENIRQ。本实验中 PE4 配置为上拉输入，无触摸时读为 1，按下时读为 0。若接错电平（接成低电平有效的从机），应将宏改为 1。

2. **CS 初始电平**：初始化时调用 `XPT2046_CS_ENABLE()` 拉低 CS 选中 XPT2046，但 XPT2046 的 DCLK 必须为低电平才开始识别 DIN 数据。如果 DCLK 拉高时 CS 拉低，可能误触发无效的 ADC 转换。本代码通过 `XPT2046_WriteCMD` 开头的 `XPT2046_CLK_LOW()` 保证安全。

3. **采样中途抬手中止**：`XPT2046_FilterXY` 在 do-while 循环中持续检查 PENIRQ，一旦抬手中止采样直接返回 -1。这种"悲观策略"避免了抬笔后的**虚假坐标**（ADC 值跳变到边界值），但需要注意：**触笔抖动**也会导致中断采样。

4. **校准参数与扫描方向绑定**：`TouchFactor[]` 中 8 套参数分别对应 LCD 的 8 种扫描方向（`LCD_SCAN_MODE 0~7`）。**修改 LCD 扫描方向后必须同步修改** `LCD_SCAN_MODE` 变量，否则触摸坐标会完全错乱。

5. **校准参数 `long double` 类型**：校准系数定义为 `long double`（80-bit 扩展精度），避免单精度浮点累积误差。`long double` 在 ARM Cortex-M4 上等价于 `double`（64-bit 双精度），GCC 中通常映射为 `__float64`，**部分编译器**（如 IAR）支持真正的 80-bit。

6. **DCLK 频率限制**：XPT2046 的 SPI 时钟最高 2.5MHz（datasheet 规定），本实验 `delay_us(5)` 在每位之间延时 10μs，对应 DCLK 频率 **50kHz**，远低于上限，**通讯稳定但速度慢**。如需快速采样，可缩短延时到 1~2μs（500kHz）。

7. **`delay_ms` / `delay_us` 软件循环**：`UTILS/delay.c` 的延时基于空循环，受编译优化级别影响，50ms 实际可能偏差 ±20%。**生产环境**应改用 SysTick 定时器或硬件定时器实现精确延时（消抖时间的精度直接受此影响）。

8. **轮询 vs 中断**：`XPT2046_DetectTouch()` 采用**轮询方式**（主循环 50ms 周期），未使用 PENIRQ 外部中断。**优点**：实现简单，无需 NVIC 配置；**缺点**：响应有 50ms 延迟，CPU 持续被占用。如需更短响应：
   - 将 PENIRQ 接 EXTI 外部中断
   - 在 EXTI 中断中读取 ADC 数据
   - 或使用 DMA 配合 SPI 硬件模块自动接收

9. **`XPT2046_CalibrationFactor` 未实现**：头文件声明了校准函数 `XPT2046_CalibrationFactor`，但 `.c` 文件中**未提供实现**。如需现场重新校准，需要自行实现：
   - 3 点校准：依次让用户触摸 LCD 屏幕的 3 个固定点
   - 读取 3 对 ADC + LCD 坐标
   - 解 6 元一次方程组（推荐高斯消元法）
   - 更新 `TouchFactor[LCD_SCAN_MODE]` 数组

10. **默认校准参数可能不准**：代码中 `TouchFactor[]` 的 8 套校准参数是**开发板出厂预设值**，仅适用于特定批次硬件。**不同 AU100 板子**（如更换 LCD 模组）需要重新校准，否则会出现：
    - 触摸点与显示光标位置**偏移**
    - 边缘区域**明显偏差**（仿射变换在外推时误差大）
    - 旋转/镜像**显示反向**

11. **坐标越界处理**：`XPT2046_GetTouchPoint` 仅做仿射变换，**未做 LCD 范围截断**。如果 ADC 值异常或校准参数不准确，可能计算出 `< 0` 或 `> 239/319` 的坐标，调用方需自行处理越界（如绘图前做 `x = max(0, min(239, x))`）。

12. **强函数与弱函数同名冲突**：本实验定义了 `XPT2046_TouchDown` 和 `XPT2046_TouchUp` 为 `__weak`，意味着**应用层可在其他 .c 文件定义同名非 weak 函数覆盖默认实现**。**注意**：编译器对 weak + 同名非 weak 的处理是**未定义行为**（部分编译器报警告），推荐用单独的回调函数指针或注册机制实现。
