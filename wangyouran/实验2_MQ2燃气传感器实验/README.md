# 实验2 — MQ2 燃气传感器实验

## 实验概述

基于 STM32F407 的 ADC1 外设读取 MQ2 燃气传感器的模拟输出电压，实现可燃气体浓度检测。MQ2 可检测甲烷、丁烷、液化石油气（LPG）、烟雾等多种可燃气体。AU100 开发板带有燃气传感器底座，支持插拔式更换不同 MQ 系列传感器。本实验通过 ADC 模数转换将传感器输出的模拟电压量化为数字量，并经 USART2 串口输出至 PC 端实时观察。

## 硬件连接

| 功能 | 引脚 | 外设 | 说明 |
|------|------|------|------|
| MQ2 模拟输出 | PC2 | ADC1 CH12 | 模拟输入，电压范围 0~3.3V |
| 调试串口 TX | PA2 | USART2 | 115200 波特率 |
| 调试串口 RX | PA3 | USART2 | 115200 波特率 |
| MQ2 加热电源 | 5V / 3.3V（板载） | — | AU100 底座供电 |

## MQ2 传感器特性

| 参数 | 说明 |
|------|------|
| 检测气体 | 甲烷（CH₄）、丁烷（C₄H₁₀）、LPG、烟雾、酒精、氢气等 |
| 输出类型 | 模拟电压（浓度越高，输出电压越高） |
| 预热时间 | 约 20~60 秒（加热丝需达到工作温度） |
| 工作电压 | 加热丝 5V，信号输出 0~VCC |
| 灵敏度 | 对 LPG、丙烷、甲烷较为灵敏 |
| 安装方式 | 传感器底座插拔式，可更换为 MQ-3 / MQ-5 / MQ-7 等系列 |

**工作原理**：MQ2 内部含加热丝和 SnO₂（二氧化锡）敏感层。加热丝将敏感层加热至工作温度后，SnO₂ 吸附空气中的氧分子导致电阻升高；遇到可燃气体时，气体与吸附氧反应释放电子，敏感层电阻下降，输出电压上升。气体浓度越高 → 电阻越低 → 输出电压越高。

## 核心模块说明

### 1. MQ2 传感器驱动 (`mq.c`)

#### GPIO 配置 (`MQ_GPIO_Config`)

PC2 配置为**模拟输入模式**（`GPIO_Mode_AIN`），这是 ADC 采集的标准做法：

```c
// PC2 → 模拟输入，无上下拉
GPIO_InitStructure.GPIO_Pin  = GPIO_Pin_2;
GPIO_InitStructure.GPIO_Mode = GPIO_Mode_AIN;
GPIO_InitStructure.GPIO_PuPd = GPIO_PuPd_NOPULL;
```

**关键点**：模拟输入模式下 GPIO 数字输入施密特触发器被关闭，可降低功耗并避免数字噪声耦合到 ADC。

#### ADC1 通用配置 (`MQ_ADC_Config` — ADC 通用寄存器)

| 参数 | 值 | 说明 |
|------|-----|------|
| `ADC_Mode` | Independent | 独立模式，仅使用 ADC1 |
| `ADC_Prescaler` | Div4 | ADC 时钟 = 84MHz / 4 = **21MHz** |
| `ADC_DMAAccessMode` | Disabled | 不使用 DMA |
| `ADC_TwoSamplingDelay` | 20 Cycles | 双 ADC 模式下采样间隔（本实验无关） |

> ADC 时钟必须在 30MHz 以内（12 位精度下推荐 ≤36MHz），21MHz 符合规范。

#### ADC1 独立配置（规则通道）

| 参数 | 值 | 说明 |
|------|-----|------|
| `ADC_Resolution` | 12b | 12 位分辨率（0~4095） |
| `ADC_ScanConvMode` | DISABLE | 单通道，不开扫描 |
| `ADC_ContinuousConvMode` | DISABLE | 单次转换，软件触发 |
| `ADC_ExternalTrigConvEdge` | None | 无外部触发，纯软件控制 |
| `ADC_DataAlign` | Right | 右对齐（低 12 位有效） |
| `ADC_NbrOfConversion` | 1 | 每次只转换一个通道 |

#### 规则通道配置

```c
ADC_RegularChannelConfig(MQ_ADC, ADC_Channel_12, 1, ADC_SampleTime_28Cycles);
//                        ADC1    通道12（PC2）     第1个序列位置   28 周期采样时间
```

- **通道 12**：对应 PC2 引脚
- **采样时间 28 周期**：在 21MHz ADC 时钟下约 1.33μs，对 MQ2 这类慢速模拟信号足够充分

### 2. ADC 采集流程 (`MQ_ReadValue`)

采用**软件触发 + 单次转换**模式，每次调用执行一次完整的 ADC 转换：

```
MQ_ReadValue():
  ├─ ADC_ClearFlag(EOC)              清除上一次转换完成标志
  ├─ ADC_SoftwareStartConv()         软件触发开始转换
  ├─ 等待 EOC 标志置位（带超时保护）
  │    └─ 最多等待 100ms，超时返回 0
  └─ ADC_GetConversionValue()        读取 12 位转换结果
```

**超时保护**：若 ADC 硬件异常导致 EOC 标志始终不置位，超时机制（100ms）可防止主循环死锁。

**为什么不用连续转换模式？**
- 本实验每 1 秒采集一次，单次转换 + 软件触发更节能
- MQ2 传感器响应速度较慢（秒级），不需要高频连续采样

### 3. ADC 值到电压的转换 (`main.c`)

```c
voltageValue = ((float)adcValue / 4096.0f) * 3.3f;
```

| 参数 | 值 | 说明 |
|------|-----|------|
| ADC 分辨率 | 12 位 | 量化范围 0 ~ 4095 |
| 参考电压 | 3.3V | STM32F407 VREF+ 连接至 3.3V |
| 除数 | 4096 | 2¹² = 4096 个量化台阶 |

> 注意除数为 **4096** 而非 4095：12 位 ADC 将 0~3.3V 分为 4096 个等间隔台阶，0 对应 0V，4095 对应 (4095/4096)×3.3V ≈ 3.299V。

**电压-浓度对应关系**（定性）：

```
0.0V ───── 空气中洁净状态（基线）
0.1~0.5V ─ 微量气体
0.5~1.5V ─ 中等浓度
1.5~3.0V ─ 高浓度 / 接近饱和
```

> 精确的 ppm 浓度计算需要根据传感器特性曲线和标定数据，本实验仅提供原始电压测量。

### 4. USART2 串口输出 (`usart.c`)

与前述实验相同的基础设施：

- `fputc` 重定向 → printf 输出到 USART2
- `fgetc` 重定向 → scanf 从 USART2 读取
- 回显中断：收到数据后原样发回
- 附带 `UART_SendByte` / `UART_SendString` / `UART_SendHalfWord` 工具函数

### 5. 主程序流程 (`main.c`)

```
上电
 │
 ├─ UART2_Init(115200)             串口初始化
 ├─ MQ_Init()                      初始化 PC2 模拟输入 + ADC1
 ├─ printf "MQ Init OK"
 │
 └─ while(1):
      ├─ adcValue = MQ_ReadValue()      读取 ADC 原始值 (0~4095)
      ├─ voltageValue = adcValue/4096*3.3   转换为电压值
      ├─ printf "AD=XXXX, X.XXXV"        串口输出当前读数
      └─ delay_ms(1000)                  等待 1 秒
```

## 运行效果

连接串口助手（115200, 8N1），上电后输出：

```
MQ Init OK
MQ2 ADC Test Start
AD=0123, 0.099V
AD=0125, 0.101V
AD=0124, 0.100V
...
```

- 无气体时：输出稳定在基线值附近（通常 AD 值较低，取决于环境）
- 有可燃气体靠近时：AD 值和电压明显上升
- 撤离气源后：数值缓慢回落（传感器恢复需要时间）

## 关键技术点

| 技术 | 说明 |
|------|------|
| MQ2 传感器 | SnO₂ 半导体气敏传感器，电阻随气体浓度变化 |
| GPIO 模拟输入 | `GPIO_Mode_AIN` 关闭数字施密特触发器 |
| ADC1 独立模式 | 单通道 CH12，12 位分辨率，软件触发 |
| ADC 时钟分频 | APB2 84MHz / 4 = 21MHz，符合 ADC 工作频率要求 |
| 单次转换 | 软件触发一次转换，非连续模式，适合低频采集 |
| 超时保护 | 等待 EOC 标志最多 100ms，异常时返回 0 |
| 电压转换 | `ADC/4096 × 3.3V`，12 位量化 |
| printf 重定向 | fputc 映射到 USART2，实现串口格式化输出 |
| 预热注意 | MQ2 需上电预热 20~60 秒后读数才稳定 |

## 代码注意事项

1. **传感器预热**：MQ2 上电后需要 20~60 秒预热，此期间读数偏高或不稳定。代码中未加入预热等待逻辑，建议上电后先空跑几十秒再观察读数，或在 `MQ_Init` 后增加 `delay_ms(60000)`。

2. **ADC 参考电压依赖板载电源质量**：电压转换使用固定 3.3V，若板载 LDO 输出有偏差（如 3.25V 或 3.35V），转换结果也会有相应误差。高精度场合建议使用外部基准电压源或先校准 VREF。

3. **`delay.h` 不在本目录**：延时函数依赖共享模块 `delay.c/h`，采用软件空循环实现，受编译优化影响。编译时需将 `7-1/basic/delay.c` 加入工程并配置 include path。

4. **输出电压范围**：MQ2 传感器加热丝需 5V 供电（由 AU100 底座提供），但信号输出电压通常设计在 0~3.3V 内以兼容 STM32 ADC 输入。若信号超过 3.3V 会损坏 MCU 引脚，产生开发板底座上的分压电路应已保证此安全限制。

5. **传感器插座兼容性**：AU100 底座支持多种 MQ 系列传感器（MQ-2 / MQ-3 / MQ-5 / MQ-7 等），更换传感器后无需修改代码，仅检测对象不同（MQ-3 侧重酒精，MQ-7 侧重 CO）。
